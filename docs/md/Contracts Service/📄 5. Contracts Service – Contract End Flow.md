# **📄 Contracts Service -- Contract End Flow**

## **1. Purpose**

The **end flow** allows a client to stop working with **one individual
talent profile** at a time.

- You cannot bulk-end a whole project/service/team at once.

- Each contract must be ended **individually**.

- Ending triggers **final billing**, **review requests**, and
  **notifications**.

## **3. Manual End Process (Step-by-Step)**

1.  **Client initiates end\**

    - Calls PUT /contracts/:id/end with reason + notes.

2.  **System validates\**

    - Contract must be ACTIVE.

    - Only the client who owns the contract can end it.

3.  **Billing finalization\**

    - Call Billing Service:

      - Close out hours logged (via Time Tracking).

      - Generate final invoice.

      - Apply credits if necessary (e.g., unused funds).

4.  **Contract status update\**

    - Mark contract ENDED.

    - Set endAtISO = now().

5.  **Review trigger\**

    - Emit contract.ended event → Review Service opens a **90-day review
      window**.

6.  **Notifications\**

    - Notify client: "Contract ended successfully."

    - Notify talent: "Your contract has been ended. Please leave a
      review."

## **4. Allowed Reasons for Ending**

The UI shows predefined reasons (from the **End Contract** design):

- **Contract successfully completed\**

- **Scope of work changed\**

- **Issues with collaboration\**

- **Service no longer needed\**

- **Other (custom note)\**

👉 These reasons should be stored in a **config table or JSON file**,
not hardcoded, so they can be updated without redeploy.

## **7. Service Logic**

File: /src/services/contracts.service.ts

import { Injectable, NotFoundException, BadRequestException } from
\'@nestjs/common\';

import { EndContractDto } from \'../dtos/end-contract.dto\';

import { ContractStatus } from \'../entities/enums\';

import { BillingClient } from \'./billing.client\';

import { ReviewsClient } from \'./reviews.client\';

import { NotificationsClient } from \'./notifications.client\';

\@Injectable()

export class ContractsService {

constructor(

private readonly billingClient: BillingClient,

private readonly reviewsClient: ReviewsClient,

private readonly notificationsClient: NotificationsClient

) {}

async endContract(id: string, dto: EndContractDto) {

// 1. Fetch contract from DB

const contract = await this.findContractById(id);

if (!contract) throw new NotFoundException(\'Contract not found\');

if (contract.status !== ContractStatus.ACTIVE) {

throw new BadRequestException(\'Only active contracts can be ended\');

}

// 2. Call Billing Service to finalize invoice/credits

const billingResult = await
this.billingClient.finalizeContract(contract.id);

// 3. Update contract status

contract.status = ContractStatus.ENDED;

contract.endAtISO = new Date().toISOString();

// save contract to DB here\...

// 4. Trigger Review Service

await this.reviewsClient.triggerReview(contract.id, contract.clientId,
contract.talentId);

// 5. Send Notifications

await this.notificationsClient.notifyContractEnded(contract,
dto.reasonCode);

return {

contractId: contract.id,

status: contract.status,

billing: billingResult,

reviewWindow: \'90 days open\'

};

}

private async findContractById(id: string) {

// Mock fetch for doc purposes

return { id, status: ContractStatus.ACTIVE, clientId: \'cli-1\',
talentId: \'tal-1\' };

}

}

## **10. Summary**

- End flow is **manual per person only**.

- Integrates with Billing to finalize payments.

- Supports standardized end reasons.

- Emits events to Review Service for 90-day window.

- Sends notifications to both client and talent.

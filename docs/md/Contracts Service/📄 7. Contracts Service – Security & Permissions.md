# **📄 Contracts Service -- Security & Permissions**

## **1. Purpose**

The Contracts Service manages sensitive operations: starting contracts
(financial implications), ending contracts (affects payments/reviews),
and accepting/rejecting offers.\
Security ensures:

- Only authorized users perform actions.

- Roles (client vs talent) are respected.

- Every action is auditable for compliance.

## **2. JWT Integration with API Gateway**

- All requests flow through the **API Gateway**.

- Gateway issues a **JWT** token after user authentication.

- Contracts Service validates JWT on every request.

- JWT contains:

{

\"sub\": \"user-123\", // userId

\"role\": \"client\|talent\", // role

\"email\": \"user@email.com\",

\"iat\": 1696159200,

\"exp\": 1696166400

}

### **Implementation**

File: /src/middleware/jwt-auth.guard.ts

import { Injectable, CanActivate, ExecutionContext,
UnauthorizedException } from \'@nestjs/common\';

import \* as jwt from \'jsonwebtoken\';

\@Injectable()

export class JwtAuthGuard implements CanActivate {

canActivate(context: ExecutionContext): boolean {

const req = context.switchToHttp().getRequest();

const authHeader = req.headers\[\'authorization\'\];

if (!authHeader) throw new UnauthorizedException(\'Missing Authorization
header\');

const token = authHeader.split(\' \')\[1\];

try {

const payload = jwt.verify(token, process.env.JWT_SECRET \|\|
\'changeme\');

req.user = payload; // attach decoded token to request

return true;

} catch (err) {

throw new UnauthorizedException(\'Invalid token\');

}

}

}

### **Apply Guard Globally**

In main.ts:

app.useGlobalGuards(new JwtAuthGuard());



## **3. Role Rules**

### **Clients**

- ✅ Can **start new contracts** (POST /contract-groups).

- ✅ Can **end contracts** they own (PUT /contracts/:id/end).

- ❌ Cannot accept/reject offers (not their role).

### **Talent**

- ✅ Can **accept or reject** contract offers (PUT
  /contracts/:id/respond).

- ❌ Cannot end contracts.

- ❌ Cannot start contracts.

### **Implementation**

File: /src/middleware/roles.guard.ts

import { Injectable, CanActivate, ExecutionContext, ForbiddenException
} from \'@nestjs/common\';

\@Injectable()

export class RolesGuard implements CanActivate {

constructor(private readonly allowedRoles: string\[\]) {}

canActivate(context: ExecutionContext): boolean {

const req = context.switchToHttp().getRequest();

const user = req.user; // decoded JWT payload

if (!this.allowedRoles.includes(user.role)) {

throw new ForbiddenException(\`Role \${user.role} is not allowed for
this action\`);

}

return true;

}

}

### **Example Usage**

****\@UseGuards(new RolesGuard(\[\'client\'\]))

\@Post(\'contract-groups\')

createGroup(@Body() dto: CreateGroupDto) {

return this.groupsService.createGroup(dto);

}

## **4. Audit Logging**

All contract changes must be recorded for compliance:

- Who did it (userId + role).

- What action was taken.

- On which contract/group.

- When it happened.

### **Audit Table**

File: /src/entities/audit.entity.ts

import { Entity, PrimaryGeneratedColumn, Column, CreateDateColumn }
from \'typeorm\';

\@Entity(\'contract_audits\')

export class ContractAudit {

\@PrimaryGeneratedColumn(\'uuid\')

id: string;

\@Column() userId: string;

\@Column() role: string;

\@Column() action: string; // e.g., \"CONTRACT_ENDED\"

\@Column({ nullable: true }) contractId: string;

\@Column({ nullable: true }) groupId: string;

\@Column(\'jsonb\', { nullable: true }) details: any;

\@CreateDateColumn()

createdAt: Date;

}

### **Audit Logger**

File: /src/services/audit-logger.service.ts

import { Injectable } from \'@nestjs/common\';

import { ContractAudit } from \'../entities/audit.entity\';

\@Injectable()

export class AuditLoggerService {

async log(user: any, action: string, details: any = {}) {

const audit = new ContractAudit();

audit.userId = user.sub;

audit.role = user.role;

audit.action = action;

audit.contractId = details.contractId \|\| null;

audit.groupId = details.groupId \|\| null;

audit.details = details;

// Save to DB

// await repo.save(audit);

console.log(\'AUDIT LOG:\', audit);

return audit;

}

}

## **5. Developer Notes**

- **Always validate JWT** → no unauthenticated requests.

- **Always enforce role guards** → clients vs talent.

- **Always log** → any lifecycle change (created, activated, ended,
  canceled).

- Logs must include enough metadata for compliance investigations.

✅ With this doc, a developer can:

- Validate JWT from API Gateway.

- Enforce role-based access control (RBAC).

- Write every contract action into an **audit trail** for compliance.

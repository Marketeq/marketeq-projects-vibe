**5. Add Credit Card & Billing Address\
\**
This guide assumes you're using Stripe Elements, and you've already
built backend APIs for card tokenization, billing address storage, and
saved card management.

## **🎯 Purpose of This Screen**

Enable the user to:

1.  Enter new credit/debit card information.

2.  Choose or enter a billing address.

3.  Save card as default if desired.

4.  Continue to payment submission.

## **🧩 Dependencies**

- **Stripe Elements** (for card fields)

- **User Service** (for address book data)

- **Checkout Service** (for card submission, saved card retrieval)

- **Session/Cart Context** (should already have price details and
  balance)

## **🗂️ Key Frontend States & Actions**

### **1. 🧾 Load Saved Billing Addresses**

**API Call**: GET /user/billing-addresses\
**Returned Fields**:

interface BillingAddress {

id: string

fullName: string

addressLine1: string

addressLine2?: string

city: string

state: string

postalCode: string

country: string

phoneNumber: string

}

- \
  Render these as selectable options.

- One can be marked default and preselected if user has one.

### **2. ➕ Add New Billing Address**

**Form Fields**:

- Full Name

- Address Line 1

- Address Line 2 (optional)

- City, State, Zip, Country

- Phone Number

**API Call**: POST /user/billing-addresses\
**Payload**:

{

fullName: string,

addressLine1: string,

addressLine2?: string,

city: string,

state: string,

postalCode: string,

country: string,

phoneNumber: string

}

✅ On success, append the new address to the list and set it as
selected.

### **3. 💳 Stripe Elements Integration**

Use CardElement to render:

- Card Number

- Name on Card

- Expiration

- CVC

**Tokenization Call**:

const {token, error} = await stripe.createToken(CardElement, {

name: \"Name from input\",

address_line1: \"Line 1\",

address_line2: \"Line 2\",

address_city: \"City\",

address_state: \"State\",

address_zip: \"Zip\",

address_country: \"Country\"

})



### **4. 💾 Save Card to Checkout Service**

Once tokenized:

**API Call**: POST /checkout/cards\
**Payload**:

{

stripeToken: string,

billingAddressId: string,

saveAsDefault: boolean

}

- Backend will store the card and return a reference ID or confirm
  success.

- If saveAsDefault is true, mark this card as default on the user
  profile.

### **5. 🗑️ Remove Existing Saved Card**

If user wants to remove a saved card:\
**API Call**: DELETE /checkout/cards/:cardId

## **🛑 Validation Requirements**

- All card fields required (handled by Stripe)

- Billing address required before saving

- Required: phone number, address line 1, city, state, zip, country

## **✅ Submission Check**

Once user enters:

- A valid credit card (tokenized)

- A selected or newly added billing address

→ Enable "Submit Order" button.

## **🔐 Notes**

- Stripe Elements auto-secures card fields --- never store raw card
  data.

- You are only saving Stripe token and metadata to the backend.

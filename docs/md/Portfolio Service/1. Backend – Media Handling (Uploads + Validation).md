# **Backend -- Media Handling (Uploads + Validation)**

Purpose: Implement secure file upload handling for the Portfolio
Service. This covers receiving files from the Editor.js Image Tool,
generating signed Cloudflare R2 URLs, validating uploads, and cleaning
up files when a portfolio item is deleted.

## **0) Prerequisites**

- Repo path: apps/portfolio-service

- Media code folder: src/media/

- Cloudflare R2 account set up

- Environment variables configured:

  - CLOUDFLARE_ACCOUNT_ID

  - CLOUDFLARE_ACCESS_KEY_ID

  - CLOUDFLARE_SECRET_ACCESS_KEY

  - CLOUDFLARE_BUCKET

  - CLOUDFLARE_PUBLIC_URL (e.g., https://cdn.example.com)

  - MAX_FILE_SIZE_MB (e.g., 10)

## **2) Media Service (Handles Uploads + Deletes)**

> File: src/media/media.service.ts

import { Injectable, BadRequestException } from \"@nestjs/common\";

import \* as AWS from \"aws-sdk\";

import { extname } from \"path\";

import { lookup as lookupMime } from \"mime-types\";

import { v4 as uuidv4 } from \"uuid\";

\@Injectable()

export class MediaService {

private s3: AWS.S3;

private bucket = process.env.CLOUDFLARE_BUCKET!;

private cdnUrl = process.env.CLOUDFLARE_PUBLIC_URL!;

private maxSize = Number(process.env.MAX_FILE_SIZE_MB \|\| 10) \* 1024
\* 1024;

constructor() {

this.s3 = new AWS.S3({

endpoint:
\`https://\${process.env.CLOUDFLARE_ACCOUNT_ID}.r2.cloudflarestorage.com\`,

accessKeyId: process.env.CLOUDFLARE_ACCESS_KEY_ID,

secretAccessKey: process.env.CLOUDFLARE_SECRET_ACCESS_KEY,

signatureVersion: \"v4\",

});

}

async generateUpload(fileName: string, fileSize: number) {

if (fileSize \> this.maxSize) {

throw new BadRequestException(\"File too large\");

}

const ext = extname(fileName).toLowerCase();

const mimeType = lookupMime(ext);

if (!mimeType \|\| !mimeType.startsWith(\"image/\")) {

throw new BadRequestException(\"Invalid file type\");

}

const key = \`portfolio/\${uuidv4()}\${ext}\`;

const url = await this.s3.getSignedUrlPromise(\"putObject\", {

Bucket: this.bucket,

Key: key,

ContentType: mimeType,

Expires: 300, // 5 minutes

});

return {

uploadUrl: url,

publicUrl: \`\${this.cdnUrl}/\${key}\`,

};

}

async deleteFile(key: string) {

await this.s3.deleteObject({ Bucket: this.bucket, Key: key }).promise();

}

}

## **3) Media Controller (Endpoints)**

> File: src/media/media.controller.ts

import { Controller, Post, Body, Delete, Query } from
\"@nestjs/common\";

import { MediaService } from \"./media.service\";

\@Controller(\"media\")

export class MediaController {

constructor(private media: MediaService) {}

\@Post(\"upload\")

async getSignedUpload(@Body() body: { fileName: string; fileSize: number
}) {

return this.media.generateUpload(body.fileName, body.fileSize);

}

\@Delete(\"remove\")

async removeFile(@Query(\"key\") key: string) {

return this.media.deleteFile(key);

}

}

## **7) Notes**

- Editor.js Image Tool should first call /media/upload with fileName +
  fileSize to get uploadUrl + publicUrl.

- Frontend then PUTs the file to uploadUrl.

- Save the returned publicUrl in your portfolio item JSON.

- Cleanup logic can be extended later to remove unused files when items
  are deleted.

With this doc, your backend developer has everything needed to implement
secure media handling for the portfolio service.

## **Step 1 --- NestJS: internal revalidation endpoint (new, tiny)**

**Explanation\**
This does **not** change your assembler or sync logic. It only
invalidates your **in-API cache** for category-page payloads, and (if
you use it) pings your **Next.js on-demand revalidate** or CDN purge.
It's safe to call many times; it dedupes. (Leave any existing cron/sync
code as-is.)

**Folder path\**
/apps/api/src/internal/revalidate.controller.ts

// /apps/api/src/internal/revalidate.controller.ts

import { Body, Controller, Headers, Post } from \'@nestjs/common\'

import { CategoryPagesCache } from \'../category-pages/cache\' // your
existing cache layer (Redis/memory)

import { CategoryPagesService } from
\'../category-pages/category-pages.service\'

type PageKind =
\'show-all\'\|\'projects\'\|\'services\'\|\'teams\'\|\'talent\'

/\*\*

\* Payload example:

\* { slugPaths: \[ \[\"software-development\"\],
\[\"software-development\",\"blockchain\",\"smart-contracts\"\] \],

\* kinds: \[\"show-all\",\"projects\"\] }

\*/

\@Controller(\'internal/revalidate\')

export class RevalidateController {

private lastHit = 0

constructor(

private readonly pagesCache: CategoryPagesCache,

private readonly pagesService: CategoryPagesService,

) {}

\@Post()

async revalidate(

\@Body() body: { slugPaths: string\[\]\[\]; kinds?: PageKind\[\] },

\@Headers(\'x-revalidate-token\') token: string

) {

if (token !== process.env.REVALIDATE_TOKEN) return { ok: false, reason:
\'unauthorized\' }

// Optional jitter/throttle to avoid storms

const now = Date.now()

const gap = Number(process.env.REVALIDATE_THROTTLE_MS \|\| 0)

if (gap && now - this.lastHit \< gap) return { ok: true, throttled: true
}

this.lastHit = now

const kinds: PageKind\[\] = (body.kinds && body.kinds.length ?
body.kinds :
\[\'show-all\',\'projects\',\'services\',\'teams\',\'talent\'\])

// 1) Invalidate API cache entries for each slugPath x kind

for (const sp of body.slugPaths) {

for (const k of kinds) {

await this.pagesCache.invalidate(sp, k) // implement to drop the cache
key

}

}

// 2) (Optional) trigger Next.js/CDN revalidate by hitting your web's
revalidate route

// If you already rely on API-only caching, skip this.

// await
fetch(\`\${process.env.PUBLIC_WEB_ORIGIN}/api/revalidate-category\`, {

// method: \'POST\',

// headers: { \'content-type\':\'application/json\',
\'x-revalidate-token\': process.env.REVALIDATE_TOKEN! },

// body: JSON.stringify({ slugPaths: body.slugPaths, kinds })

// })

return { ok: true, count: body.slugPaths.length, kinds }

}

}

> If you already have a similar internal endpoint, **don't duplicate
> it**---just accept the slugPaths/kinds payload format and handle cache
> invalidation there.

## **Step 3 --- NestJS helpers to compute affected slug paths (small, additive)**

**Explanation\**
These helpers **reuse your repos/services** to compute the precise list
of category page paths to revalidate. No assembler changes. QA docs
already rely on breadcrumb/tree endpoints; we expose two small "affected
pages" endpoints for **tags** and **tax groups**.

**Folder path\**
/apps/api/src/categories/affected.controller.ts (new)

import { Controller, Get, Param } from \'@nestjs/common\'

import { CategoriesService } from \'./categories.service\'

import { TaxGroupsService } from \'../tax-groups/tax-groups.service\'

\@Controller()

export class AffectedController {

constructor(

private readonly categories: CategoriesService,

private readonly taxGroups: TaxGroupsService,

) {}

\@Get(\'/categories/:uuid/graph\')

async graph(@Param(\'uuid\') uuid: string) {

// Return canonical slugPath plus all descendants\' slugPaths

const node = await this.categories.findByUuid(uuid)

if (!node) return { slugPath: \[\], descendants: \[\] }

const slugPath = await this.categories.slugPathFor(uuid) // e.g.,
\[\'software-development\',\'blockchain\',\'smart-contracts\'\]

const descendants = await this.categories.descendantSlugPaths(uuid) //
string\[\]\[\]

return { slugPath, descendants }

}

\@Get(\'/tags/:uuid/affected-category-slugs\')

async affectedByTag(@Param(\'uuid\') uuid: string) {

const cats = await this.categories.findCategoriesLinkedToTag(uuid) //
uuids of categories

const slugPaths = await this.categories.slugPathsForUuids(cats)

return { slugPaths }

}

\@Get(\'/tax-groups/:uuid/affected-category-slugs\')

async affectedByRule(@Param(\'uuid\') uuid: string) {

// Resolve which categories include this rule in their block set

const catUuids = await this.taxGroups.categoryUuidsUsingRule(uuid)

const slugPaths = await this.categories.slugPathsForUuids(catUuids)

return { slugPaths }

}

}

> This follows your existing sync and tree helpers; we're just exposing
> "which pages are affected?" in a compact way for Strapi to call.

## **Step 5 --- Cron fallback (reuse what you have; add revalidation sweep)**

**Explanation\**
You already run a **10-minute cron** to sync missed updates. We add a
**light sweep** that looks for recently updated
categories/tags/tax-groups and calls **Revalidate** once. Keep your
existing cron; just append the sweep.

**Folder path\**
/apps/api/src/cron/revalidate.cron.ts (new, additive)

import { Cron, CronExpression } from \'@nestjs/schedule\'

import { Injectable } from \'@nestjs/common\'

import { CategoriesService } from \'../categories/categories.service\'

\@Injectable()

export class RevalidateCron {

private lastRun = 0

constructor(private readonly categories: CategoriesService) {}

\@Cron(CronExpression.EVERY_10_MINUTES)

async sweep() {

// If webhooks failed, revalidate anything touched since last run

const since = new Date(this.lastRun \|\| Date.now() - 11 \* 60 \* 1000)

this.lastRun = Date.now()

const changed = await this.categories.changedSince(since) // returns
category uuids and maybe tag/tax-group impacts

if (!changed?.length) return

const slugPaths = await this.categories.slugPathsForUuids(changed)

await fetch(process.env.REVALIDATE_BASE!, {

method: \'POST\',

headers: { \'content-type\':\'application/json\',
\'x-revalidate-token\': process.env.REVALIDATE_TOKEN! },

body: JSON.stringify({ slugPaths })

})

}

}

> Your QA docs already validate cron fallback flows (disable webhook,
> wait, verify). This just layers revalidation into that same safety
> net.

## **Step 6 --- QA: Postman + manual (add to existing collections)**

**Explanation\**
Extend your existing **Category Sync QA** with three quick revalidation
checks.

1.  **Category update triggers revalidate\**

    - Update name/slug or publish flag in Strapi.

    - Verify POST /internal/revalidate received with the correct
      slugPaths.

    - Fetch /api/category-pages/:slug → blocks/SEO reflect immediately.

2.  **Tag update triggers revalidate\**

    - Update tag name or assignment on items that power a block.

    - Verify affected category pages revalidated via
      /tags/:uuid/affected-category-slugs.

3.  **Tax group change triggers revalidate\**

    - Edit a "Trending/Top-Rated/Newest" rule.

    - Verify impacted pages revalidated via
      /tax-groups/:uuid/affected-category-slugs.

4.  **Cron fallback\**

    - Temporarily disable Strapi→NestJS webhook.

    - Make a change; wait 10 min; confirm pages updated post-cron.

## **What we explicitly did not rebuild**

- Your existing **cron** sync job. (We only appended a tiny

- Your Category Page Assembler/Tax Groups/SEO logic. (We only purge

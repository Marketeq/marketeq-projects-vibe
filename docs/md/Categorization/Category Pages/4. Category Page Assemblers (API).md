# **Doc 4 --- Category Page Assemblers (API)**

L1/L2/L3 Payloads (Show‑All / Projects / Services / Teams / Talent)

**Goal:** Create a unified API (/api/category-pages/:slug) that
assembles all blocks (Show‑All, Projects, Services, Teams, Talent, plus
optional Guides & Courses editorial blocks) into a consistent JSON
response for category pages.

Note: This rewrite removes any reference to a separate
"Similarity/Suggestions service." "Related Categories" is produced
**locally** via the Hugging Face embeddings component described in Doc
10. All other parts remain unchanged unless explicitly noted below.

**Category Pages module lives inside the Categories microservice**

**Step 1 --- API Contract (unchanged keys + suggestions)**

**Explanation\**
Each category page payload contains:

- **level:** L1 / L2 / L3

- **slugPath:** array of slugs, e.g.
  \[\"software-development\",\"blockchain\",\"smart-contracts\"\]

- **uuids:** L1, L2, L3 UUIDs for DB queries

- **breadcrumbs:** full path hierarchy

- **seo:** title/description/canonical (hybrid SEO)

- **hero:** optional hero section (curated editorial, pulled from
  Strapi)

- **subcategories:** children categories of this node

- **blocks:** dynamic content blocks from Tax Groups (Projects,
  Services, Teams, Talent, etc.)

- **pagination:** cursor metadata for "load more"

- **suggestions:** suggestion modules (Related/Trending/Popular
  categories, Related skills)

## **Step 3 --- Service & Assembler**

### **3.1 Service (build page payload)**

**Explanation\**
The service resolves the category (level/uuids/breadcrumbs/published),
collects subcategories, fetches Tax Groups (rules), composes the blocks
via evaluators, and then builds **suggestions** using the **local
Hugging Face embeddings helper** (no external similarity service).

**Folder path\**
/apps/api/src/category-pages/category-pages.service.ts

import { Injectable } from \'@nestjs/common\'

import { CategoryRepo } from \'../repos/category.repo\'

import { TaxGroupRepo } from \'../repos/taxgroup.repo\'

import { Assembler } from \'./assembler\'

import { getSeoForPath } from \'../seo/seo\'

import { PagePayload, CategoryPageSuggestions } from \'./types\'

import { buildSuggestions } from \'./suggestions/local\' // local HF
helper (Doc 10)

\@Injectable()

export class CategoryPagesService {

constructor(

private readonly categories: CategoryRepo,

private readonly taxGroups: TaxGroupRepo,

private readonly assembler: Assembler

) {}

async buildPagePayload(

slugPath: string\[\],

kind: \'show-all\'\|\'projects\'\|\'services\'\|\'teams\'\|\'talent\' =
\'show-all\'

): Promise\<PagePayload \| null\> {

const resolved = await this.categories.resolvePath(slugPath)

if (!resolved \|\| !resolved.published) return null

const subcategories = await
this.categories.childrenOf(resolved.uuids\[\`l\${resolved.level}\`\])

const rules = await this.taxGroups.forPath(resolved.uuids,
resolved.level)

const blocks = await this.assembler.composeBlocks(resolved.uuids,
resolved.level, rules, kind)

// Suggestions via local embeddings + repo heuristics

const suggestions: CategoryPageSuggestions = await buildSuggestions({

categoriesRepo: this.categories,

currentUuid: resolved.uuids\[\`l\${resolved.level}\`\],

l1Slug: slugPath\[0\] \|\| \'\',

limit: { related: 24, trending: 24, popular: 24, skills: 30 }

})

const seo = await getSeoForPath(slugPath)

return {

level: resolved.level,

slugPath,

uuids: resolved.uuids,

breadcrumbs: resolved.breadcrumbs,

seo,

hero: resolved.hero \|\| null,

subcategories,

blocks,

pagination: undefined,

suggestions

}

}

}

### **3.2 Assembler (evaluators & blocks)**

**Folder path\**
/apps/api/src/category-pages/assembler.ts

import { Injectable } from \'@nestjs/common\'

import { EvaluatorFactory } from \'./evaluators/factory\'

import { BlockSpec } from \'./types\'

\@Injectable()

export class Assembler {

constructor(private readonly factory: EvaluatorFactory) {}

async composeBlocks(

uuids: { l1: string; l2?: string; l3?: string },

level: 1\|2\|3,

rules: any\[\],

kind: \'show-all\'\|\'projects\'\|\'services\'\|\'teams\'\|\'talent\' =
\'show-all\'

): Promise\<BlockSpec\[\]\> {

const blocks: BlockSpec\[\] = \[\]

for (const rule of rules) {

if (rule.pageKind && rule.pageKind !== kind) continue

const evaluator = this.factory.forEntity(rule.entity, rule.kind)

const items = await evaluator.run({ uuids, level, rule, limit:
rule.limit })

// Carousel guard: require at least 3× grid size

if (rule.carousel && items.length \< (rule.gridSize \* 3)) continue

blocks.push({

key: \`rule-\${rule.id}\`,

type: rule.carousel ? \'carousel\' : \'grid\',

entity: rule.entity,

items: items.slice(0, rule.limit),

pagination: items.length === rule.limit

? { nextCursor: items\[items.length-1\].uuid }

: undefined

})

}

return blocks

}

}

## **Step 4 --- Hero / Editorial Blocks (from Strapi)**

**Explanation\**
Some L1/L2 pages (per design) may include a hero banner or editorial
sections (Guides, Courses). These can be authored in Strapi and injected
into the payload.

**Strapi example (attributes):**

****{

\"hero_title\": { \"type\": \"string\" },

\"hero_image\": { \"type\": \"media\" },

\"hero_link\": { \"type\": \"string\" }

}

The assembler sets these into payload.hero when present.

## **Step 5 --- Pagination (cursors, no duplicates)**

- Return nextCursor in a block when the evaluator indicates more
  results.

- Frontend may call the same endpoint with ?cursor=\... to fetch next
  items for that block.

- Prevents endless carousel scrolling; cap carousel scrolls to your
  product limit (e.g., 10 pages max).

**Snippet:**

****pagination: items.length === rule.limit

? { nextCursor: items\[items.length-1\].uuid }

: undefined

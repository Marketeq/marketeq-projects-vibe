# **09 - Strapi Admin Guide -- Publishing Categories to the Marketplace**

## **Purpose**

This guide explains how to decide when a new category or subcategory is
ready to be **published in the marketplace**. The goal is to ensure that
categories appear with enough content to make the marketplace look full
and professional.

## **1. When to Publish a Category**

- **New categories** (created by users, AI, or admins) are not
  automatically shown in the marketplace.

- Categories must first build up enough projects, jobs, or services to
  create a **strong visual grid**.

- You, as the admin, decide when a category is ready to be published by
  manually reviewing search results.

## **2. How to Review a Category**

1.  Log in to the **Strapi admin panel**.

2.  Navigate to **Content Manager → Categories**.

3.  Open the category or subcategory you want to review.

4.  Search for it in the marketplace frontend.

5.  Look at the search result grid:

    - Aim for at least **2--3 full rows of content** (e.g., 6, 8, 9, or
      12 items).

    - Make sure the category page doesn't look empty or sparse.

## **3. Publishing the Category**

- If the grid looks good:

  - In Strapi, set Published in Marketplace = true.

  - Save changes.

  - The category will now appear publicly and generate its own category
    page.

- If the grid looks too empty:

  - Leave the category unpublished.

  - Check back later when more content has been added.

## **4. Important Notes**

- There is **no automatic threshold** for publishing.

- You control when a category is ready based on **visual review**, not
  on a set number of items.

- A category can still be used internally and appear in autocomplete
  suggestions before being published.

## **5. Best Practices**

- Always check **subcategories** as well as top-level categories.

- Keep unpublished categories hidden until they create a strong first
  impression.

- Use the "Created At" filter in Strapi to easily find new categories
  that need review.

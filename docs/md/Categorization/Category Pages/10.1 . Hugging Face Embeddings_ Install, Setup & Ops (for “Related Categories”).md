# **Doc 10.1 --- Hugging Face Embeddings: Install, Setup & Ops (for "Related Categories")**

We use **sentence-transformers/all-MiniLM-L6-v2** (384-dim) to compute
embeddings for each category (name + short description + ancestors).
Vectors live in Postgres via **pgvector**. Category pages query nearest
neighbors by cosine similarity.\
This is **offline precompute** (no per-request model calls). No HF
account or API key required for public models.

## **Step 0 --- Choose install path (pick ONE)**

- **Path A (recommended): Offline worker** that encodes categories and
  stores vectors in Postgres (pgvector). Cheapest, fastest, most stable.

- **Path B (optional): Managed HF Inference Endpoint** for real-time
  (skip unless you truly need on-demand embedding). If you want this,
  see "Appendix B".

Everything below assumes **Path A**.

CREATE EXTENSION IF NOT EXISTS vector;

\-- Side table for 384-d embeddings (MiniLM)

CREATE TABLE IF NOT EXISTS category_embedding (

category_uuid UUID PRIMARY KEY REFERENCES categories(uuid) ON DELETE
CASCADE,

embed vector(384) NOT NULL,

updated_at timestamptz NOT NULL DEFAULT now()

);

\-- Approximate nearest neighbors index for cosine distance

\-- Tune lists based on row count (start at \~rows/1000)

CREATE INDEX IF NOT EXISTS idx_category_embed_ivf

ON category_embedding USING ivfflat (embed vector_cosine_ops) WITH
(lists = 100);

import os, sys, time

import psycopg2

import psycopg2.extras

from sentence_transformers import SentenceTransformer

DB = os.environ\[\"DATABASE_URL\"\]

MODEL_NAME = os.getenv(\"MODEL_NAME\",
\"sentence-transformers/all-MiniLM-L6-v2\")

BATCH_SIZE = int(os.getenv(\"BATCH_SIZE\", \"256\"))

def fetch_rows(conn):

\# Get categories whose updated_at \> embedding.updated_at, or missing
embedding

sql = \"\"\"

SELECT c.uuid,

c.name,

COALESCE(c.short_description, \'\') AS short_description,

\-- slugs for ancestors: adjust to your schema

l1.slug AS l1, l2.slug AS l2, l3.slug AS l3

FROM categories c

LEFT JOIN categories l3 ON l3.uuid = c.uuid

LEFT JOIN categories l2 ON l2.uuid = c.parent_uuid

LEFT JOIN categories l1 ON l1.uuid = l2.parent_uuid

WHERE c.updated_at \> COALESCE(

(SELECT updated_at FROM category_embedding e WHERE e.category_uuid =
c.uuid),

\'1970-01-01\'

)

\"\"\"

with conn.cursor(cursor_factory=psycopg2.extras.DictCursor) as cur:

cur.execute(sql)

return cur.fetchall()

def upsert(conn, uuid, vec):

with conn.cursor() as cur:

cur.execute(

\"\"\"

INSERT INTO category_embedding(category_uuid, embed, updated_at)

VALUES (%s, %s, NOW())

ON CONFLICT (category_uuid)

DO UPDATE SET embed=EXCLUDED.embed, updated_at=NOW()

\"\"\",

(uuid, list(vec)) \# pgvector accepts Python lists

)

def main():

conn = psycopg2.connect(DB)

conn.autocommit = True

rows = fetch_rows(conn)

if not rows:

print(\"No categories need (re)embedding.\")

return

model = SentenceTransformer(MODEL_NAME)

batch = \[\]

for row in rows:

path = \" \> \".join(x for x in \[row\[\"l1\"\], row\[\"l2\"\],
row\[\"l3\"\]\] if x)

text = f\"{path} --- {row\[\'name\'\]}\"

if row\[\"short_description\"\]:

text += f\" --- {row\[\'short_description\'\]}\"

batch.append((row\[\"uuid\"\], text))

if len(batch) \>= BATCH_SIZE:

uuids, texts = zip(\*batch)

embs = model.encode(list(texts), convert_to_numpy=True,
normalize_embeddings=True)

for uid, vec in zip(uuids, embs):

upsert(conn, uid, vec)

batch = \[\]

if batch:

uuids, texts = zip(\*batch)

embs = model.encode(list(texts), convert_to_numpy=True,
normalize_embeddings=True)

for uid, vec in zip(uuids, embs):

upsert(conn, uid, vec)

print(\"Embeddings updated successfully.\")

if \_\_name\_\_ == \"\_\_main\_\_\":

main()

## **Step 3 --- API repo helpers (pgvector queries)**

**Explanation\**
Add helpers used by Doc 10's SuggestionsService to fetch embeddings and
nearest neighbors.

**Folder path\**
/apps/api/src/categories/categories.repo.ts

export class CategoriesRepo {

constructor(private readonly db: {
query\<T=any\>(sql:string,params?:any\[\]):Promise\<{rows:T\[\]}\> }) {}

async nodeFor(ctx: { l1?: string; l2?: string; l3?: string }) {

const slug = ctx.l3 \|\| ctx.l2 \|\| ctx.l1

if (!slug) return null

const { rows } = await this.db.query(\`SELECT uuid, slug, l1_slug FROM
categories WHERE slug = \$1 LIMIT 1\`, \[slug\])

return rows\[0\] \|\| null

}

async embeddingFor(uuid: string) {

const { rows } = await this.db.query(\`SELECT embed FROM
category_embedding WHERE category_uuid = \$1\`, \[uuid\])

return rows\[0\] \|\| null

}

async nearestNeighbors(opts: { uuid: string; l1Slug: string; limit:
number }) {

const { uuid, l1Slug, limit } = opts

const { rows } = await this.db.query(\`

WITH focus AS (SELECT embed FROM category_embedding WHERE category_uuid
= \$1)

SELECT c.uuid, c.name, c.slug, c.parent_uuid, p.parent_uuid AS
parent_of_parent,

c.published_in_marketplace,

ce.embed \<=\> (SELECT embed FROM focus) AS cos_dist

FROM categories c

JOIN category_embedding ce ON ce.category_uuid = c.uuid

LEFT JOIN categories p ON p.uuid = c.parent_uuid

WHERE c.published_in_marketplace = true

AND c.uuid \<\> \$1

AND c.l1_slug = \$2

ORDER BY cos_dist ASC

LIMIT \$3

\`, \[uuid, l1Slug, limit\])

return rows

}

async nearestNeighborsGlobal(opts: { uuid: string; limit: number }) {

const { uuid, limit } = opts

const { rows } = await this.db.query(\`

WITH focus AS (SELECT embed FROM category_embedding WHERE category_uuid
= \$1)

SELECT c.uuid, c.name, c.slug, c.parent_uuid, p.parent_uuid AS
parent_of_parent,

c.published_in_marketplace,

ce.embed \<=\> (SELECT embed FROM focus) AS cos_dist

FROM categories c

JOIN category_embedding ce ON ce.category_uuid = c.uuid

LEFT JOIN categories p ON p.uuid = c.parent_uuid

WHERE c.published_in_marketplace = true

AND c.uuid \<\> \$1

ORDER BY cos_dist ASC

LIMIT \$2

\`, \[uuid, limit\])

return rows

}

}

## **Step 4 --- Suggestions service (wire "Related Categories" to embeddings)**

**Explanation\**

async related(ctx: { l1?: string; l2?: string; l3?: string }) {

const focus = await this.repo.nodeFor(ctx)

if (!focus) return \[\]

const embed = await this.repo.embeddingFor(focus.uuid)

if (!embed) return \[\]

let rows = await this.repo.nearestNeighbors({ uuid: focus.uuid, l1Slug:
focus.l1_slug, limit: 24 })

if (rows.length \< 8) {

const extra = await this.repo.nearestNeighborsGlobal({ uuid: focus.uuid,
limit: 24 })

const seen = new Set(rows.map(r =\> r.uuid))

rows.push(\...extra.filter(e =\> !seen.has(e.uuid)))

}

return Promise.all(rows.map(r =\> this.toSuggestedCategory(r)))

}

## **Step 5 --- Triggers: when do embeddings refresh?**

**Explanation\**
You already send Strapi→NestJS webhooks for category changes. You
**don't** need a new endpoint. The worker reads categories.updated_at;
any row newer than its last embedding is re-embedded.

**Do this:**

- After Strapi saves a category (create/update/move), it already updates
  categories.updated_at → the next worker run picks it up.

- Schedule the worker to run every **5--10 minutes** (or faster if you
  like), and at deploy time.

## **Step 6 --- Ops & Perf**

- **Memory/CPU:** MiniLM encodes fast on CPU. Start with **1 vCPU / 1--2
  GB RAM**.

- **Index tuning:** Increase IVFFlat lists as row count grows (rebuild
  index off-hours).

- **Normalization:** We set normalize_embeddings=True---cosine distance
  is stable.

- **Backfill:** On first deploy, run the worker once without cron to
  fill all vectors.

- **Monitoring:** Track COUNT(\*) FROM category_embedding vs COUNT(\*)
  FROM categories (should match for published rows); alarm on big gaps.

## **Step 8 --- Rollback plan**

- If anything goes sideways, drop the suggestions from the payload (UI
  hides the section), and/or DROP INDEX idx_category_embed_ivf; DROP
  TABLE category_embedding; (your main category system remains intact).

- Since this is additive, rollback risk is low.

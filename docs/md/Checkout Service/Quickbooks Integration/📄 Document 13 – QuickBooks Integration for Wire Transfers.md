# **📄 Document 13 -- QuickBooks Integration for Wire Transfers**

### **🧭 Purpose**

This document provides full implementation instructions for integrating
wire transfer payments with QuickBooks, as wire transfers are not
processed through Stripe. It covers how invoices are generated by the
platform, how they are synced with QuickBooks, and how payment status is
confirmed through QuickBooks to finalize the checkout flow.

Wire transfers are only supported for **payment in full** at this time.

## **🏗️ Microservice**

This logic resides in the accounts-receivable-service, which is
responsible for:

- Creating wire transfer invoices in the platform database

- Syncing invoice data to QuickBooks

- Verifying payment status via QuickBooks API

- Marking projects/contracts as "Paid" when confirmed

- Triggering confirmation emails, Slack alerts, and internal logs

## **🔁 Wire Transfer Flow Overview**

1.  **User completes checkout** → selects wire transfer as the payment
    method

2.  **Platform generates invoice** → invoice is created and stored in
    Postgres, marked unpaid

3.  **Invoice synced to QuickBooks** → includes all required
    contract/project metadata

4.  **Admin confirms receipt in QuickBooks** manually

5.  **Platform polls QuickBooks** periodically or on webhook

6.  **If paid** → project/contract marked as active, payment badge added

7.  **If failed or missing** → triggers retry logic and notifies
    accounts receivable

## **⚙️ Invoice Generation (Platform)**

Invoices are created by the platform **at the time of checkout**, even
before payment is received.

## **🔄 Sync to QuickBooks**

### **✅ Required Setup**

- OAuth2 connected app credentials

- QuickBooks account ID and customer mapping

- Line items are not broken out for wire transfers (single lump sum
  only)

### **📤 Sync API Logic**

****//
src/accounts-receivable-service/integrations/quickbooks/quickbooks.service.ts

public async createQuickbooksInvoice(invoice: PlatformInvoiceDTO) {

const customer = await
this.findOrCreateQuickbooksCustomer(invoice.clientId);

const quickbooksInvoice = {

CustomerRef: { value: customer.Id },

TxnDate: new Date().toISOString().split(\'T\')\[0\],

DueDate: invoice.dueDate,

Line: \[

{

Amount: invoice.amount,

DetailType: \'SalesItemLineDetail\',

Description: \`\${invoice.contractType.toUpperCase()} --
\${invoice.contractName}\`,

SalesItemLineDetail: {

ItemRef: { value: \'1\', name: \'Service\' } // default service item

}

}

\]

};

const result = await
this.quickbooksApi.createInvoice(quickbooksInvoice);

return result;

}

## **🔎 Verifying Wire Payment Status**

Polling or webhook-based confirmation (depending on QuickBooks plan):

public async verifyQuickbooksInvoicePaymentStatus(invoiceId: string):
Promise\<boolean\> {

const qbInvoice = await this.quickbooksApi.getInvoiceById(invoiceId);

if (qbInvoice?.Balance === 0) {

await this.markPlatformInvoiceAsPaid(invoiceId);

await this.triggerSuccessFlow(invoiceId);

return true;

}

return false;

}

## **🔒 Safety Features**

- Only platform triggers invoice creation (no manual endpoint access)

- OAuth2 scopes restrict QuickBooks access to invoices only

- Logs written for every invoice sync attempt

- No developer can mark invoices as "paid" --- status must be verified
  via QuickBooks

- Slack + email alerts on sync failure or invoice mismatch

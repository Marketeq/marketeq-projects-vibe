## **1. Purpose**

Contracts don't always follow the happy path. We need to handle:

- Failed deposits

- Expired durations

- Disputed contracts

- Multi-contract projects with partial terminations

This ensures consistency across Checkout, Billing, Time Tracking,

## **2. Failed Deposits**

### **Rule**

- Contracts start in PENDING after group creation.

- They only move to ACTIVE after the deposit clears.

- If deposit fails:

  - Contracts remain PENDING.

  - Retry window (configurable, e.g., **7 days**).

  - If no retry â†’ auto-CANCELED.

### **Implementation**

## **4. Disputed Contracts**

### **Rule**

- A client or talent may **dispute** hours, payments, or deliverables.

- When disputed:

  - Contract status â†’ DISPUTED (temporary).

  - Billing pauses.

  - Time tracking locked.

  - Admin/Support Service required to resolve.

## **5. Multiple Contracts per Project (Partial Terminations)**

### **Rule**

- A project/service/team may have **N contracts**.

- Ending one contract does **not** end the others.

- Group stays active until **all contracts end**.

- Dashboard should still show project as "Active" if at least one
  contract remains.

### **Implementation**

When client ends one contract:

- Mark only that contract as ENDED.

- Emit contract.ended.

- Check if group has remaining ACTIVE contracts:

  - If yes â†’ group stays active.

  - If no â†’ emit contract.group.ended event.

Service logic:

î°ƒasync endContract(contractId: string) {

await this.repo.save(contract);

const group = await this.repo.findGroupById(contract.groupId);

const activeContracts = group.contracts.filter(c =\> c.status ===
\'active\');

if (activeContracts.length === 0) {

// all ended â†’ emit group ended

this.eventBus.publish(\'contract.group.ended\', { groupId: group.id });

}

}

## **6. Developer Notes**

- **Deposits:** Always atomic. If one fails, group blocked. If timeout
  expires â†’ cancel all.

- **Durations:** Default policy = manual renewal (avoid auto-ending to
  prevent accidental work stoppage).

- **Disputes:** Must pause billing + time tracking. Support/Admin
  required.

- **Partial terminations:** End contracts one at a time. Group ends only
  when last contract ends.

# **Doc 11 --- Schema Enforcement & Validation**

Prevent circular references, guarantee unique slugs, and maintain UUIDs
across systems (for L1/L2/L3 page assemblers)

**Applies to:** Strapi (admin) + NestJS API + Postgres\
**Goal:** enforce invariants at the DB layer first, then mirror in
Strapi lifecycles and NestJS services, so page assembly never breaks.

> **Assembler-first posture (added):** The Category Pages **assembler**
> must validate at runtime even if upstream already enforces these
> rules. If your DB/Strapi already have the constraints, **honor
> existing rules and skip the optional setup** below. The runtime checks
> protect the page render path without changing upstream schemas.

## **Invariants (what we're enforcing)**

1.  **No cycles in the category tree:** a node's parent_uuid can never
    be its own descendant (or itself).

2.  **Slug uniqueness (case-insensitive):\**

    - **Root (L1):** each root slug unique globally.

    - **Children (L2/L3):** slug must be unique within the same parent
      (sibling uniqueness).

3.  **Stable UUIDs across systems:** the same uuid identifies a category
    in Strapi and in the NestJS DB; upserts are keyed by uuid, not by
    slug.

## **Step 0 --- Assembler Validation Contract (runtime-only; always on) *(added)***

**Why:** Page assembly must be safe even if an upstream system
regresses. These checks run on every request; they **do not** modify
upstream schemas.

**Folder path\**

export type Node = { uuid: string; slug: string; parentUuid:
string\|null; published: boolean }

export function assertPath(resolved: Node\[\]): { level: 1\|2\|3 } {

if (!resolved.length) throw make404(\'Category not found\')

// Published-only: every node in the chain must be published

if (!resolved.every(n =\> n.published)) throw make404(\'Category
unpublished\')

// No cycles + proper ancestry

const seen = new Set\<string\>()

for (let i = resolved.length - 1; i \>= 0; i\--) {

const n = resolved\[i\]

if (seen.has(n.uuid)) throw make500(\'Cycle detected in category
chain\')

seen.add(n.uuid)

const parent = resolved\[i - 1\]

if (i \> 0 && parent.uuid !== (n.parentUuid \|\| \'\')) {

throw make500(\'Broken ancestry: parent UUID mismatch\')

}

}

const level = (resolved.length as 1\|2\|3)

if (level \< 1 \|\| level \> 3) throw make404(\'Unsupported depth\')

return { level }

}

export function dedupeSiblingsBySlug\<T extends { slug: string }\>(rows:
T\[\]): T\[\] {

const seen = new Set\<string\>()

const out: T\[\] = \[\]

for (const r of rows) {

const k = r.slug.trim().toLowerCase()

if (!seen.has(k)) { seen.add(k); out.push(r) }

}

return out

}

// helpers

function make404(msg: string) { const e = new Error(msg); (e as
any).status = 404; return e }

function make500(msg: string) { const e = new Error(msg); (e as
any).status = 500; return e }

**Use in the assembler/service (example hook):**

****import { assertPath, dedupeSiblingsBySlug } from \'./validation\'

const resolved = await this.categories.resolvePath(slugPath) //
downstream API result: \[L1, L2?, L3?\]

const { level } = assertPath(resolved) // runtime assertions only (no
schema mods)

const uuids = { l1: resolved\[0\].uuid, l2: resolved\[1\]?.uuid, l3:
resolved\[2\]?.uuid }

const subcategoriesRaw = await
this.categories.childrenOf(uuids\[\`l\${level}\`\])

const subcategories = dedupeSiblingsBySlug(subcategoriesRaw) //
presentation guard

> **Error mapping (runtime)**:\
> • 404 --- category not found or unpublished\
> • 500 --- cycle/broken ancestry (should be impossible if upstream
> enforces)

## **Step 1 --- Postgres: extensions + columns + indexes (authoritative)**

**Why:** put guardrails in the database so no app can violate
invariants.\
**Optional if already enforced upstream.** *(label added; content
unchanged)*

**Folder path\**
/apps/api/src/db/migrations/2025_10_02_schema_enforcement.sql

**SQL**

****\-- 1) Extensions

CREATE EXTENSION IF NOT EXISTS citext; \-- case-insensitive text

CREATE EXTENSION IF NOT EXISTS ltree; \-- hierarchical path ops

\-- 2) Columns we rely on (adapt if they already exist)

\-- slug_ci is case-insensitive; path stores UUID lineage

L1.L2.L3 as ltree

ALTER TABLE categories

ALTER COLUMN slug TYPE citext USING slug::citext;

ALTER TABLE categories

ADD COLUMN IF NOT EXISTS path ltree;

\-- 3) Slug uniqueness:

\-- Root (parent_uuid IS NULL): unique slug among roots

CREATE UNIQUE INDEX IF NOT EXISTS ux_categories_root_slug

ON categories (slug)

WHERE parent_uuid IS NULL;

\-- Children (parent_uuid NOT NULL): unique per parent + slug

CREATE UNIQUE INDEX IF NOT EXISTS ux_categories_child_slug

ON categories (parent_uuid, slug)

WHERE parent_uuid IS NOT NULL;

\-- 4) Optional: prevent duplicate (uuid,parent) pairs (usually PK

already covers uuid)

\-- (Skip if uuid is the primary key)

\-- 5) Path index to speed ancestor/descendant checks

CREATE INDEX IF NOT EXISTS ix_categories_path_gist

ON categories USING GIST (path);

\-- 6) Trigger function to compute path and prevent cycles

CREATE OR REPLACE FUNCTION

categories_set_path_and_prevent_cycles()

RETURNS trigger AS \$\$

DECLARE

parent_path ltree;

BEGIN

\-- Compute new path:

IF NEW.parent_uuid IS NULL THEN

\-- Root: path = own uuid

NEW.path := (NEW.uuid::text)::ltree;

ELSE

\-- Child: path = parent.path \|\| own uuid

SELECT path INTO parent_path FROM categories WHERE uuid =

NEW.parent_uuid;

IF parent_path IS NULL THEN

RAISE EXCEPTION \'Parent % not found or has no path\',

NEW.parent_uuid;

END IF;

\-- Prevent cycle: parent path must NOT contain the child

itself

IF parent_path @\> (NEW.uuid::text)::ltree THEN

RAISE EXCEPTION \'Cycle detected: cannot set parent of % to

its descendant %\', NEW.uuid, NEW.parent_uuid;

END IF;

NEW.path := parent_path \|\| (NEW.uuid::text)::ltree;

END IF;

RETURN NEW;

END;

\$\$ LANGUAGE plpgsql;

\-- 7) BEFORE INSERT/UPDATE trigger attaches the path and checks

cycles

DROP TRIGGER IF EXISTS trg_categories_set_path ON categories;

CREATE TRIGGER trg_categories_set_path

BEFORE INSERT OR UPDATE OF parent_uuid, uuid

ON categories

FOR EACH ROW

EXECUTE FUNCTION categories_set_path_and_prevent_cycles();

**Result:**

- Cycles impossible: trying to set a node's parent to itself/a
  descendant raises an error.

- Slug collisions impossible: enforced by unique indexes (root-level and
  per-sibling).

- Case-insensitive: citext makes "Blockchain" and "blockchain" the same
  slug.

export function normalizeSlug(input: string): string {

return input

.trim()

.toLowerCase()

.replace(/\[\'\"\]/g, \'\') // strip quotes

.replace(/\[\^a-z0-9\]+/g, \'-\') // non-alnum -\> dash

.replace(/\^-+\|-+\$/g, \'\') // trim dashes

.slice(0, 120); // safety cap

}

normalizeSlug(dto.name)

name: dto.name,

slug,

} catch (e: any) {

\|\| \'\')) {

throw new Error(\`Slug \'\${slug}\' already exists at this level. Choose
a different name.\`)

}

if (/Cycle detected/i.test(e.message)) {

throw new Error(\`Invalid parent: circular reference is not allowed.\`)

}

throw e

}

}

}

## **Step 3 --- Strapi: lifecycles to preserve UUID + sibling-slug check (additive)**

**Why:** when an editor creates/edits a category, we ensure the UUID
stays stable, and we don't allow sibling slug collisions before the
change leaves Strapi.\
**Optional if already enforced via your model/UID field.** *(label
added; content unchanged)*

**Folder path\**
/strapi/src/api/category/content-types/category/lifecycles.js

const { v4: uuidv4 } = require(\'uuid\')

module.exports = {

async beforeCreate(event) {

const data = event.params.data

// 1) Ensure UUID exists (preserve if provided by import/sync)

if (!data.uuid) data.uuid = uuidv4()

// 2) Normalize slug (uid field may exist, but normalize anyway)

if (!data.slug && data.name) data.slug = slugify(data.name)

if (data.slug) data.slug = slugify(data.slug)

// 3) Sibling slug guard (case-insensitive) --- prevent editor confusion
early

const parent = data.parent_uuid \|\| null

const conflict = await
strapi.db.query(\'api::category.category\').findOne({

where: {

parent_uuid: parent,

slug: data.slug,

},

select: \[\'id\'\],

})

if (conflict) {

throw new Error(\`Slug \'\${data.slug}\' already exists under this
parent.\`)

}

},

async beforeUpdate(event) {

const { data, where } = event.params

// Never allow UUID changes via admin edits

if (data.uuid) delete data.uuid

// Normalize slug if editing name/slug

if (data.name && !data.slug) data.slug = slugify(data.name)

if (data.slug) data.slug = slugify(data.slug)

// Sibling slug guard

if (data.slug \|\| data.parent_uuid !== undefined) {

const parent = data.parent_uuid ?? undefined

const existing = await
strapi.db.query(\'api::category.category\').findOne({

where: { id: where.id },

select: \[\'uuid\', \'parent_uuid\'\],

})

const targetParent = (parent === undefined) ? existing.parent_uuid :
parent

const conflict = await
strapi.db.query(\'api::category.category\').findOne({

where: {

id: { \$ne: where.id },

parent_uuid: targetParent,

slug: data.slug,

},

select: \[\'id\'\],

})

if (conflict) throw new Error(\`Slug \'\${data.slug}\' already exists
under this parent.\`)

}

},

}

function slugify(s) {

return (s \|\| \'\')

.toString()

.trim()

.toLowerCase()

.replace(/\[\'\"\]/g, \'\')

.replace(/\[\^a-z0-9\]+/g, \'-\')

.replace(/\^-+\|-+\$/g, \'\')

.slice(0, 120)

}

## **Step 4 --- Cross-system UUID discipline (idempotent sync)**

**Why:** page assemblers must rely on UUIDs, not slugs. Make sync
idempotent by UUID from both directions. *(unchanged)*

**Contract (no rebuild, just clarity):**

- Create/Update (either direction): upsert by uuid.

- Rename/move: slug or parent change does not change uuid.

- Delete: send uuid to delete; consumers should purge by uuid.

## **Step 5 --- Page Assembler guard (resolve by slug path, return UUID chain)**

**Why:** URLs use slugs, but render by UUIDs. Resolution validates the
chain (no gaps; correct parentage) and returns the stable UUIDs for
blocks/queries. *(unchanged)*

**Folder path\**
/apps/api/src/category-pages/resolve.service.ts

import { Injectable } from \'@nestjs/common\'

import { CategoriesRepo } from \'../categories/categories.repo\'

\@Injectable()

export class CategoryPathResolver {

constructor(private readonly repo: CategoriesRepo) {}

// Accepts \[\'/l1\',\'l2?\',\'l3?\'\] slugs; returns { level, uuids,
breadcrumbs } or null

async resolve(slugPath: string\[\]) {

// 1) Fetch nodes by slug in order and verify parent chain

const \[l1, l2, l3\] = slugPath

const n1 = l1 ? await this.repo.findBySlugAndParent(l1, null) : null

if (!n1) return null

const n2 = l2 ? await this.repo.findBySlugAndParent(l2, n1.uuid) : null

if (l2 && !n2) return null

const n3 = l3 ? await this.repo.findBySlugAndParent(l3, n2!.uuid) : null

if (l3 && !n3) return null

const level = l3 ? 3 : l2 ? 2 : 1

const uuids = { l1: n1.uuid, \...(n2 && { l2: n2.uuid }), \...(n3 && {
l3: n3.uuid }) }

const breadcrumbs = \[

{ name: n1.name, slug: n1.slug, uuid: n1.uuid },

\...(n2 ? \[{ name: n2.name, slug: n2.slug, uuid: n2.uuid }\] : \[\]),

\...(n3 ? \[{ name: n3.name, slug: n3.slug, uuid: n3.uuid }\] : \[\]),

\]

return { level, uuids, breadcrumbs, published: \[n1, n2,
n3\].filter(Boolean).every(n =\> (n as any).published_in_marketplace) }

}

}

/apps/api/src/categories/categories.repo.ts (helpers)

async findBySlugAndParent(slug: string, parent_uuid: string \| null) {

const { rows } = await this.db.query(\`

SELECT uuid, name, slug, parent_uuid, published_in_marketplace

FROM categories

WHERE slug = \$1 AND

((parent_uuid IS NULL AND \$2::uuid IS NULL) OR parent_uuid = \$2::uuid)

LIMIT 1

\`, \[slug, parent_uuid\])

return rows\[0\] \|\| null

}

## **Step 7 --- Ops notes**

*(unchanged)*

- **UUID continuity:** never regenerate UUIDs---treat as primary
  identity across systems.

- **Slug changes:** allowed, but do not mint new UUIDs. Route resolution
  handles new slugPath; canonical URLs update via SEO builder.

- **Soft delete:** if you use deleted_at, consider partial unique
  indexes to ignore soft-deleted rows (add WHERE deleted_at IS NULL to
  the two unique indexes).

- **Depth:** if you want to hard-cap depth at L3, add a CHECK:
  nlevel(path) \<= 3.

## **Step 8 --- "Skip if you already have it"**

*(unchanged, and now emphasized up top)*

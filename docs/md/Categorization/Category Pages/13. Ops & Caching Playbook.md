# **Ops & Caching Playbook**

**Goal:** make category pages (L1/L2/L3) fast and stable with
predictable cache keys, safe purges, and proactive monitoring---without
touching anything already built in the category service.

## **1) Cache-Key Strategy**

### **1.1 URL-level caching (CDN/default)**

CDN caches by **URL**. Our page routes already vary by URL:

- /c/:l1

- /c/:l1/:l2

- /c/:l1/:l2/:l3

- Plus ?kind=all\|projects\|services\|teams\|talent and pagination
  cursors where applicable.

You don't need to add these into custom keys; they're inherently part of
the URL.

### **1.2 Surrogate keys (for precise purge)**

We'll tag each response with *surrogate keys* so we can purge all
affected variants in one shot.

**Key dimensions (required):**

- **Level**: l1\|l2\|l3

- **UUID path**: l1_uuid, optional l2_uuid, optional l3_uuid

- **Rule hash**: fingerprint of the rule inputs used to assemble blocks
  (Tax Group rules + config + model/version flags)

**Canonical surrogate key (joined by spaces):**

****cp:v1

lvl:{1\|2\|3}

l1:{UUID}

l2:{UUID?}

l3:{UUID?}

rule:{RULE_HASH}

**Optional dimensions (append as needed):**

- kind:{all\|projects\|services\|teams\|talent}

- locale:{xx-YY}

- model:{REL_CAT_EMB_V} (semantic "Related Categories" model/data
  version)

## **2) API Headers & TTLs**

**File:** /apps/api/src/category-pages/category-pages.controller.ts (or
your existing controller)

import { Controller, Get, Param, Query, Res } from \'@nestjs/common\'

import type { Response } from \'express\'

import { createHash } from \'crypto\'

import { CategoryPagesService } from \'./category-pages.service\'

function ruleHash(input: unknown) {

// stable JSON + SHA1 → short base16

const json = JSON.stringify(input, Object.keys(input as any).sort())

return createHash(\'sha1\').update(json).digest(\'hex\').slice(0, 12)

}

\@Controller(\'/api/category-pages\')

export class CategoryPagesController {

constructor(private readonly svc: CategoryPagesService) {}

\@Get(\[\'/:l1\', \'/:l1/:l2\', \'/:l1/:l2/:l3\'\])

async get(

\@Param(\'l1\') l1: string,

\@Param(\'l2\') l2: string,

\@Param(\'l3\') l3: string,

\@Query(\'kind\') kind:
\'all\'\|\'projects\'\|\'services\'\|\'teams\'\|\'talent\' = \'all\',

\@Res() res: Response

) {

const slugPath = \[l1, l2, l3\].filter(Boolean) as string\[\]

const payload = await this.svc.buildPagePayload(slugPath, kind)

if (!payload) return res.status(404).end()

// 1) Compute rule hash from inputs used to assemble blocks

const rh = ruleHash({

rules: payload.ruleInputs, // whatever you fed into the assembler

suggestionsModel: process.env.REL_CAT_EMB_V \|\| \'v1\',

kind

})

// 2) Surrogate keys

const keys = \[

\'cp:v1\',

\`lvl:\${payload.level}\`,

\`l1:\${payload.uuids.l1}\`,

payload.uuids.l2 ? \`l2:\${payload.uuids.l2}\` : null,

payload.uuids.l3 ? \`l3:\${payload.uuids.l3}\` : null,

\`rule:\${rh}\`,

\`kind:\${kind}\`,

\].filter(Boolean).join(\' \')

// 3) TTLs (edge-cache) --- tune as needed

// s-maxage: edge TTL; stale-while-revalidate lets CDN serve old content
while we refresh

res.setHeader(\'Cache-Control\', \'public, s-maxage=900,
stale-while-revalidate=86400\')

// 4) Tag response for precise purge

// Use header your CDN understands:

// - Fastly: Surrogate-Key

// - Cloudflare: cache tags via API (see purge section)

// - Akamai: CP codes / cache tags equivalent

res.setHeader(\'Surrogate-Key\', keys)

// 5) ETag for conditional requests (optional)

res.setHeader(\'ETag\', \`\"\${rh}-\${payload.checksum}\"\`)

return res.json(payload)

}

}

**Notes**

- ruleInputs: pass the exact, normalized structure the assembler used
  (Tax Group config + feature flags + caps). If it changes, the
  rule:{hash} changes → a fresh cache entry.

- Keep **published-only** enforcement in the assembler; cache won't save
  you from bad inputs.

## **3) Purge on Webhook (Selective Busting)**

When relevant data changes, **don't** shotgun purge by URL. Purge by
**surrogate keys** (tags) and **warm** the critical pages.

### **3.1 Webhook → Purge Target**

**File:** /apps/api/src/ops/purge.controller.ts

import { Body, Controller, Post } from \'@nestjs/common\'

import { CdnPurger } from \'./purger\'

import { ImpactGraph } from \'./impact-graph\'

type CategoryEvent = {

type:
\'category.updated\'\|\'category.published\'\|\'category.moved\'\|\'category.deleted\'\|\'suggestions.model.bumped\'

uuids: { l1: string; l2?: string; l3?: string }\[\]

}

\@Controller(\'/ops\')

export class PurgeController {

constructor(

private readonly purger: CdnPurger,

private readonly impact: ImpactGraph

) {}

\@Post(\'purge-category-pages\')

async purge(@Body() evt: CategoryEvent) {

// Compute affected paths (L1/L2/L3 + page kinds) from uuids

const affected = await this.impact.categoryPages(evt.uuids)

// 1) Build surrogate keys to purge

const tags = new Set\<string\>()

for (const a of affected) {

tags.add(\'cp:v1\')

tags.add(\`lvl:\${a.level}\`)

tags.add(\`l1:\${a.uuids.l1}\`)

if (a.uuids.l2) tags.add(\`l2:\${a.uuids.l2}\`)

if (a.uuids.l3) tags.add(\`l3:\${a.uuids.l3}\`)

}

// Rule hash varies → purge by broader tags so all variants go

await this.purger.purgeByTags(\[\...tags\])

// 2) Optional: warm key pages (show-all + top kinds)

await this.purger.warm(affected.flatMap(a =\> a.urlsToWarm))

return { ok: true, purgedTags: \[\...tags\] }

}

}

**File:** /apps/api/src/ops/impact-graph.ts

export class ImpactGraph {

// Returns affected page specs for each node (current page + ancestors)

async categoryPages(nodes: { l1: string; l2?: string; l3?: string }\[\])
{

const out: Array\<{ level:1\|2\|3; uuids: any; urlsToWarm: string\[\]
}\> = \[\]

for (const node of nodes) {

// 1) Current node page

out.push({

level: node.l3 ? 3 : node.l2 ? 2 : 1,

uuids: node,

urlsToWarm: this.urls(node)

})

// 2) Ancestors (breadcrumb parents)

if (node.l3) out.push({ level: 2, uuids: { l1: node.l1, l2: node.l2 },
urlsToWarm: this.urls({ l1: node.l1, l2: node.l2 }) })

out.push({ level: 1, uuids: { l1: node.l1 }, urlsToWarm: this.urls({ l1:
node.l1 }) })

}

// Deduplicate by uuid triple

const seen = new Set\<string\>()

return out.filter(x =\> {

const k =
\`\${x.uuids.l1}:\${x.uuids.l2\|\|\'\'}:\${x.uuids.l3\|\|\'\'}:\${x.level}\`

if (seen.has(k)) return false

seen.add(k); return true

})

}

urls(node: { l1: string; l2?: string; l3?: string }) {

const base = \`/c/\${\[node.l1, node.l2,
node.l3\].filter(Boolean).join(\'/\')}\`

return \[

\`\${base}?kind=all\`,

\`\${base}?kind=projects\`,

\`\${base}?kind=services\`,

\`\${base}?kind=teams\`,

\`\${base}?kind=talent\`

\]

}

}

### **3.2 CDN Purger**

**File:** /apps/api/src/ops/purger.ts

import fetch from \'node-fetch\'

export class CdnPurger {

// Example: Fastly surrogate keys; adapt to your CDN

async purgeByTags(tags: string\[\]) {

if (!tags.length) return

const chunks = chunk(tags, 50) // CDN header limits

for (const c of chunks) {

await fetch(process.env.FASTLY_API_URL!, {

method: \'POST\',

headers: {

\'Fastly-Key\': process.env.FASTLY_API_TOKEN!,

\'Surrogate-Key\': c.join(\' \'),

}

})

}

}

async warm(urls: string\[\]) {

const origin = process.env.FRONTEND_ORIGIN \|\|
\'https://www.example.com\'

const list = Array.from(new Set(urls)).slice(0, 50) // sanity cap

await Promise.all(list.map(u =\> fetch(origin + u, { method: \'GET\'
})))

}

}

function chunk\<T\>(arr: T\[\], n: number): T\[\]\[\] {

const out: T\[\]\[\] = \[\]

for (let i=0;i\<arr.length;i+=n) out.push(arr.slice(i, i+n))

return out

}

**Webhook sources that MUST hit POST /ops/purge-category-pages:**

- Category create/update/publish/move/delete

- Rule changes impacting blocks (Tax Groups)

- **Semantic related** data/model bump (set REL_CAT_EMB_V=new AND send
  event)

## **4) Cron Fallback (if webhooks fail)**

**Job:** /ops/cron/revalidate-category-pages (every 10--15 min)

- Diff updated_at in categories/rules since last run.

- Build impacted set via ImpactGraph.categoryPages.

- Call purgeByTags + warm.

This keeps pages from going stale even if a webhook drops.

## **6) Cache Warming Strategy**

After any purge:

1.  Warm **show-all** and the **most trafficked kinds**
    (projects/services) for each affected slug path---limit to first 50
    URLs per event to avoid thundering herds.

2.  Pre-warm **L1 and L2 ancestors** (cheap wins).

3.  Optionally schedule a **background sweeper** to warm L3s overnight
    based on traffic logs.

## **7) TTL Guidance & Bypass**

- **API pages (assembler JSON):** s-maxage=900,
  stale-while-revalidate=86400

- **HTML pages (Next.js):** match or slightly longer if using ISR
  (ensure revalidate hooks are wired)

- **Bypass:** respect Cache-Control: no-cache on admin-preview or
  ?preview=1 routes (serve uncached).

## **8) Model/Data Versioning for "Related Categories"**

- Set REL_CAT_EMB_V=YYYYMMDD (or semantic version) in env.

- Include it in ruleHash inputs and add model:{REL_CAT_EMB_V} as a
  surrogate tag when you need targeted purges.

- **Small edits** (single category rename): webhook targets its page +
  ancestors → minimal purge.

- **Global model bump**: increment REL_CAT_EMB_V, fire a single purge by
  tag model:{REL_CAT_EMB_V_OLD} (or purge all cp:v1 during a maintenance
  window).

## **9) Runbook (When Things Go Weird)**

- **Symptom:** page shows stale data\
  **Action:** call /ops/purge-category-pages with the node's uuids;
  confirm CDN purge 2xx; check warm logs.

- **Symptom:** suggestion modules empty\
  **Action:** check Prometheus category_page_block_empty_total with
  labels → identify block & level; verify upstream rule inputs; ensure
  "published-only" not filtering everything; validate embeddings version
  if it's a related block.

- **Symptom:** purge storms\
  **Action:** debounce webhooks (coalesce events for 2--5s) and cap
  per-minute purges. Use tag-based purges, not URL wildcards.

## **10) Quick Checklist (Wire This, Nothing Else)**

- Controller sets Cache-Control, Surrogate-Key, ETag.

- Surrogate keys follow: cp:v1 lvl:{n} l1:{uuid} \[l2:\] \[l3:\]
  rule:{hash} kind:{pageKind}.

- Purge endpoint receives webhooks → purges tags → warms URLs.

- Cron fallback runs every 10--15 min.

- Telemetry emits block_fill metrics; alerts configured for
  empty/underfilled blocks.

- REL_CAT_EMB_V included in rule hash for related-categories changes.

This keeps the **Category Pages** fast, purgeable, and
observable---strictly per designs, fully in sync with (but not
re-implementing) anything already built.

# **External APIs Call Map --- Category Pages (L1/L2/L3)**

### **Service Boundaries (Category Pages)**

- **Listing Service** --- provides items for blocks (**projects,
  services, teams**). Category Pages only calls its read endpoints; it
  does **not** implement listing logic.

- **User Service** --- provides **talent profiles** for Talent blocks.
  Called only for read/hydration when needed.

- **Categories Service** --- authoritative tree (L1/L2/L3), slugs/UUIDs,
  children, published state. Used for path resolution and subcategory
  grids.

- **Strapi CMS** --- admin authoring for taxonomy groups/rules (+
  optional hero/editorial). Category Pages reads rules; Strapi webhooks
  handle sync already.

- **Search Service** --- **out of scope** for this sprint (no search UI
  here).

- **Similarity (Related Categories)** --- there is **no separate
  "Similarity service."** For this sprint, similarity is **local**
  (Hugging Face + pgvector) inside Category Pages; if an owning service
  exposes "related" endpoints elsewhere, we may consume them as generic
  **downstream APIs**.

- **Autocomplete** --- **not used** in this sprint.

> Inputs/outputs for Related Categories (local): **input** current
> category UUID (+ optional L1 scope, limit); **output** published
> category UUIDs. If empty, omit the module.

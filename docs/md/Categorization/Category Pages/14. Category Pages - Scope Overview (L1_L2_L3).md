**Category Pages - Scope Overview (L1/L2/L3)**

**Assumptions (already exist; do not touch):**

- Category service & sync, data models, and admin workflows.

- Existing content entities (Projects/Services/Teams/Talent) and rule
  evaluators.

- Next.js app baseline and routing.

**You must build (from the designs):**

1.  **Suggestion modules** on L1/L2/L3 pages

    - Related Categories (semantic; HF embeddings).

    - Trending Categories.

    - Popular Categories.

    - Related Skills.

    - Only **published** categories render on public pages.

2.  **Page payloads & API** (Assembler)

    - Add a suggestions object to /api/category-pages/:slugPath
      responses (no breaking changes).

    - Keep "All / Projects / Services / Teams / Talent" as separate
      routes per design.

3.  **Hybrid SEO for L1/L2/L3\**

    - Default templates + overrides (use existing fields); JSON-LD
      breadcrumbs, canonical, OG/Twitter.

4.  **Selective revalidation hooks** for category-page changes (design
    modules & SEO only).

5.  **Testing & QA** (this sprint only): Postman, SQL (read-only),
    Cypress---focused **solely on category pages**.

- **Out of scope (do NOT build / do NOT describe):\**

    or sync pipelines.

  - Autocomplete or search UI.

  - Rewriting existing rule evaluators beyond what the designs require
    for the page.

- **In scope:** only what's listed above, exactly as the designs show.

    links for every L1/L2/L3 page.

    matches the design's modules and caps.

4.  **SEO** defaults and overrides render correctly (title/desc, JSON-LD

5.  **Selective revalidation** updates only the affected pages.

6.  **Cypress** suite passes for navigation, suggestions, SEO, and no
    broken links.

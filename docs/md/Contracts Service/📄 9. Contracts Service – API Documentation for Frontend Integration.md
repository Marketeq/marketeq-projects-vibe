# **📄 Contracts Service -- API Documentation for Frontend Integration**

## **1. Purpose**

This document is written for **frontend engineers**. It explains exactly
how to call the Contracts Service endpoints, what request/response
payloads look like, and where to place them in the repo.

By the end, a frontend dev can connect UI screens (like **Hire Me**,
**End Contract**, **Contract Summary**) directly to the Contracts API
without guessing.

## **2. Repo Structure (Frontend Placement)**

For a **Next.js frontend** (React):

/frontend

/services

contracts.api.ts \<- API wrapper functions

The frontend should use **fetch/axios** wrappers calling the Contracts
Service (port 3008 or via API Gateway).

## **3. Authentication**

All calls must include the **JWT token** from the API Gateway.

const headers = {

\'Content-Type\': \'application/json\',

\'Authorization\': \`Bearer \${token}\`

};



## **4. API Endpoints**

### **4.1 Create a Contract Group (Hire Multiple Talent)**

**Endpoint**

****POST /contract-groups

**Request**

****{

\"type\": \"project\",

\"ownerClientId\": \"cli-123\",

\"projectId\": \"proj-456\",

\"contracts\": \[

{

\"talentId\": \"tal-1\",

\"title\": \"UX Designer\",

\"rate\": 85,

\"schedule\": \"part-time\",

\"duration\": \"6 months\"

},

{

\"talentId\": \"tal-2\",

\"title\": \"Frontend Developer\",

\"rate\": 70,

\"schedule\": \"full-time\",

\"duration\": \"6 months\"

}

\]

}

**Response**

****{

\"groupId\": \"grp-789\",

\"contracts\": \[

{ \"id\": \"c1\", \"talentId\": \"tal-1\", \"status\": \"pending\" },

{ \"id\": \"c2\", \"talentId\": \"tal-2\", \"status\": \"pending\" }

\]

}

👉 Frontend should redirect client to **Checkout Service** for deposit
payment.

### **4.2 Accept / Reject Offer (Talent Side)**

**Endpoint**

****PUT /contracts/:id/respond

**Request**

****{

\"accepted\": true,

\"talentId\": \"tal-1\"

}

**Response**

****{

\"id\": \"c1\",

\"message\": \"Offer accepted. Waiting for deposit.\"

}



### **4.3 End a Contract (Client Side)**

**Endpoint**

****PUT /contracts/:id/end

**Request**

****{

\"reasonCode\": \"SERVICE_NO_LONGER_NEEDED\",

\"notes\": \"Work wrapped up earlier than expected.\"

}

**Response**

****{

\"contractId\": \"c1\",

\"status\": \"ended\",

\"billing\": { \"finalInvoice\": \"INV-123\", \"credit\": 50.0 },

\"reviewWindow\": \"90 days open\"

}

👉 After ending, frontend should **redirect client to Review screen**
to leave feedback for talent.

### **4.7 Webhooks (Backend → Contracts)**

> ⚠️ These are **not called by frontend**, but frontend devs should know
> when to expect state changes.

- POST /webhooks/checkout/deposit-cleared → Marks contracts active.

- POST /webhooks/billing/finalized → Updates contract billing history.

## **5. Frontend API Wrapper Example**

File: /frontend/services/contracts.api.ts

import axios from \'axios\';

const BASE_URL = process.env.API_GATEWAY_URL + \'/contracts-service\';

export const ContractsAPI = {

async createGroup(token, payload) {

const res = await axios.post(\`\${BASE_URL}/contract-groups\`, payload,
{

headers: { Authorization: \`Bearer \${token}\` }

});

return res.data;

},

async respondToOffer(token, contractId, accepted) {

const res = await
axios.put(\`\${BASE_URL}/contracts/\${contractId}/respond\`,

{ accepted },

{ headers: { Authorization: \`Bearer \${token}\` } }

);

return res.data;

},

async endContract(token, contractId, reasonCode, notes) {

const res = await
axios.put(\`\${BASE_URL}/contracts/\${contractId}/end\`,

{ reasonCode, notes },

{ headers: { Authorization: \`Bearer \${token}\` } }

);

return res.data;

},

async getContract(token, contractId) {

const res = await axios.get(\`\${BASE_URL}/contracts/\${contractId}\`, {

headers: { Authorization: \`Bearer \${token}\` }

});

return res.data;

},

async getClientContracts(token, clientId) {

const res = await
axios.get(\`\${BASE_URL}/contracts/client/\${clientId}\`, {

headers: { Authorization: \`Bearer \${token}\` }

});

return res.data;

},

async getTalentContracts(token, talentId) {

const res = await
axios.get(\`\${BASE_URL}/contracts/talent/\${talentId}\`, {

headers: { Authorization: \`Bearer \${token}\` }

});

return res.data;

}

};



## **6. Mapping UI Screens → API Calls**

- **Hire Me screen** (Hire Me.pdf) → POST /contract-groups

- **End Contract screen** (End Contract.pdf) → PUT /contracts/:id/end

- **Contract Summary** → GET /contracts/:id

- **Client Dashboard (Active/Completed Contracts)** → GET
  /contracts/client/:id

- **Talent Dashboard** → GET /contracts/talent/:id

- **Review Request Flow** (after end) → Triggered automatically by
  contract.ended event

## **7. Developer Notes**

- Always pass **JWT token** from Gateway.

- Use **idempotency keys** when creating contract groups (to avoid
  duplicates if user retries checkout).

- Expect **status transitions**:

  - pending → waiting for deposit

  - active → after deposit cleared

  - ended → when client ends

  - canceled → if deposit fails/expired

## **1) Canonical URL normalization (SSR)**

**What:** make sure /c/:l1/:l2?/:l3? and the **separate pages**
(/projects\|/services\|/teams\|/talent) always resolve to one canonical
path.

**Where:** your existing /apps/web/pages/c/\[\...slug\].tsx, inside
getServerSideProps.

î°ƒexport const getServerSideProps: GetServerSideProps = async (ctx) =\> {

const slug = (ctx.params?.slug as string\[\]) \|\| \[\]

const pageKinds = \[\'projects\',\'services\',\'teams\',\'talent\'\] as
const

const last = slug\[slug.length - 1\]

const isKind = pageKinds.includes(last as any)

const slugPath = isKind ? slug.slice(0, -1) : slug

const pageKind = (isKind ? last : \'show-all\') as
\'show-all\'\|\'projects\'\|\'services\'\|\'teams\'\|\'talent\'

const url =
\`\${process.env.NEXT_PUBLIC_API_URL}/category-pages/\${slugPath.join(\'/\')}\${pageKind===\'show-all\'
? \'\' : \`/\${pageKind}\`}\`

const res = await fetch(url)

if (res.status === 404) return { notFound: true }

const payload = await res.json()

const expected = \`/c/\${slugPath.join(\'/\')}\${pageKind===\'show-all\'
? \'\' : \`/\${pageKind}\`}\`

if (ctx.resolvedUrl !== expected) {

return { redirect: { destination: expected, permanent: true } }

}

return { props: { payload, pageKind } }

}

## **4) Page-kind links (All / Projects / Services / Teams / Talent)**

**What:** keep your existing UI; ensure these are **links to separate

**Where:** inline on the page or as

## **7) Quick QA checklist**

- Canonical redirects work (no dupes like trailing slashes or wrong
  page-kind position).

- \<SeoHead\> renders title/description, canonical, OG/Twitter, and
  JSON-LD breadcrumbs.

- Breadcrumb UI links walk the path correctly.

- **All/Projects/Services/Teams/Talent** are separate pages (SSR each).

- robots.txt/sitemap wired (optional) and correct.

### **TL;DR**

Keep your current frontend. Just add:

- \<SeoHead\>

- \<Breadcrumbs\>

- Page-kind links as **real links\**

- SSR canonical normalization

- (Optional) sitemap/robots

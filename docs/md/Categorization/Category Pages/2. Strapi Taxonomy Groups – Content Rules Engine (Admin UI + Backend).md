# **Doc 2 --- Strapi Taxonomy Groups**

**Content Rules Engine (Admin UI + Backend)**

Goal: Allow Strapi admins to define **dynamic rules (Tax Groups)** for
how content appears on L1/L2/L3 category pages. Rules can say things
like "Top Rated Projects in Blockchain Development" or "Newest Services
in Smart Contracts."

Strapi will store these rules, manage overrides, and expose them via
API/webhooks so NestJS can assemble category pages dynamically.

## **Step 1 --- Create the Taxonomy Group content-type**

**Explanation\**
A **Taxonomy Group** ties a set of rules to a specific category level
(L1, L2, or L3). For example:

- "Blockchain Development" (L2) → has Tax Groups like *Trending Teams*
  or *Newest Services*.

We store them in Strapi so admins can see/edit rules without coding.

**Folder path\**
/src/api/taxonomy-group/content-types/taxonomy-group/schema.json

**Code**

****{

\"kind\": \"collectionType\",

\"collectionName\": \"taxonomy_groups\",

\"info\": {

\"singularName\": \"taxonomy-group\",

\"pluralName\": \"taxonomy-groups\",

\"displayName\": \"Taxonomy Group\"

},

\"options\": { \"draftAndPublish\": true },

\"attributes\": {

\"title\": { \"type\": \"string\", \"required\": true },

\"category\": {

\"type\": \"relation\",

\"relation\": \"manyToOne\",

\"target\": \"api::category.category\",

\"inversedBy\": \"taxonomy_groups\"

},

\"level\": {

\"type\": \"enumeration\",

\"enum\": \[\"L1\", \"L2\", \"L3\"\],

\"required\": true

},

\"rules\": {

\"type\": \"json\",

\"required\": true,

\"default\": \[\]

},

\"manual_enabled\": { \"type\": \"boolean\", \"default\": false }

}

}

## **Step 2 --- Define the Rule schema (JSON field)**

**Explanation\**
The rules JSON holds the actual logic. Each rule describes *what to
pull* and *how to display it*. Example:

\[

{

\"id\": \"rule-001\",

\"entity\": \"projects\",

\"kind\": \"top-rated\",

\"limit\": 12,

\"gridSize\": 3,

\"carousel\": true,

\"manualEnabled\": false

}

\]

**Folder path\**
Already inside the taxonomy-group schema under \"rules\": { \"type\":
\"json\" }.

**Rule schema fields**

- id: unique string (UUID or short slug)

- entity: \"projects\" \| \"services\" \| \"teams\" \| \"talent\"

- kind: \"top-rated\" \| \"trending\" \| \"newest\" (extendable)

- limit: number of items to fetch

- gridSize: items per row (used to calculate carousel thresholds)

- carousel: boolean (true = carousel, false = grid)

- manualEnabled: boolean (allows pinned overrides from Strapi)

**Explanation\**
Whenever a Taxonomy Group is updated, we log it for traceability (to
match your reassignment audit approach).

**Folder path\**
/src/api/taxonomy-group/content-types/taxonomy-group/lifecycles.js

**Code**

****module.exports = {

## **Step 6 --- Expose Tax Groups via API**

**Explanation\**
NestJS will call Strapi to fetch rules for a given category. We'll
create a secure endpoint.

**Folder path\**
/src/api/taxonomy-group/controllers/taxonomy-group.js

**Code**

****\'use strict\';

module.exports = {

async findByCategory(ctx) {

const { uuid } = ctx.params;

const groups = await
strapi.db.query(\'api::taxonomy-group.taxonomy-group\').findMany({

where: { category: { uuid } },

populate: true

});

ctx.body = groups;

}

};

**Route\**
/src/api/taxonomy-group/routes/taxonomy-group.js

module.exports = {

routes: \[

{

method: \'GET\',

path: \'/taxonomy-groups/category/:uuid\',

handler: \'taxonomy-group.findByCategory\',

config: {

auth: { scope: \[\'admin\'\] }

}

}

\]

};

## **Step 7 --- Sync to NestJS**

**Explanation\**
When a Taxonomy Group is created/updated/deleted, Strapi fires a webhook
to NestJS to refresh cache.

**Folder path\**
/src/api/taxonomy-group/content-types/taxonomy-group/lifecycles.js

**Code (append to lifecycle)**

****async afterCreate(event) {

const { result } = event;

await
strapi.plugins\[\'webhook\'\].services.webhook.trigger(\'taxonomy-group.changed\',
{

id: result.id,

category: result.category,

rules: result.rules

});

},

async afterDelete(event) {

const { result } = event;

await
strapi.plugins\[\'webhook\'\].services.webhook.trigger(\'taxonomy-group.deleted\',
{

id: result.id

});

}



## **Step 8 --- Testing**

**Checklist**

- Create a Taxonomy Group in Strapi linked to an L1 category.

- Add a rule JSON:
  \"entity\":\"projects\",\"kind\":\"top-rated\",\"limit\":6.

- Save → webhook fires to NestJS.

- Call API in NestJS: GET /category-pages/software-development → should
  include the new block in the payload.

- Update rule → NestJS payload changes on next call.

- Delete group → payload excludes block.

- Confirm audit log entry created.

✅ With this, Strapi now has a **Taxonomy Groups engine**:

- Admins define rules (Projects, Services, Teams, Talent).

- Each L1/L2/L3 category can have multiple Tax Groups.

- Rules support manual overrides, carousels, and AI-driven ranking.

- Events sync to NestJS in real-time.

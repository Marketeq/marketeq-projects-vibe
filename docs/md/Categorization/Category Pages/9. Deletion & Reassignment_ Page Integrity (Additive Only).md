# **Doc 9 --- Deletion & Reassignment: Page Integrity (Additive Only)**

**Goal:** When a parent category is deleted (or moved), make sure:

- **Tax Groups** (your rules that power "Top-Rated/Trending/Newest",
  etc.) automatically **retarget** away from the removed subtree.

- Editors see an **impact preview** and must confirm retargeting.

- NestJS and Strapi **stay in sync**.

- Category pages ship with **no broken/empty blocks** after the change.

> What you already have (no rebuild):\
> • Forced reassignment UX in Strapi + plugin flow (modal + block if no
> new parent) and NestJS sync hooks.\
> • Content reassignment logic on delete in NestJS (roll-up to parent
> when needed).\
> • QA cases for "Content Reassignment on Deletion."\
> • High-level delete/rename handling requiring reassignment.

Everything below is **new** and copy-paste ready.

## **A) Auto-Retarget Tax Groups (NEW)**

**Explanation\**
If a Tax Group rule targets a subtree that includes the **deleted/moved
parent**, automatically retarget the rule to the **new parent** chosen
in your reassignment step (or roll up to ancestor if that was the
confirmed action). This prevents rules from pointing to nothing and
avoids empty blocks.

**Folder path\**
/apps/api/src/tax-groups/tax-groups.service.ts (additive)

// /apps/api/src/tax-groups/tax-groups.service.ts

import { Injectable } from \'@nestjs/common\'

import { PrismaService } from \'../db/prisma.service\'

type RetargetPlan = {

ruleId: string

oldTarget: { uuid: string, path: string\[\] }

newTarget: { uuid: string, path: string\[\] }

}

\@Injectable()

export class TaxGroupsService {

constructor(private readonly prisma: PrismaService) {}

async affectedBySubtree(rootUuid: string): Promise\<{ ruleId: string,
targetUuid: string }\[\]\> {

// Find any rules whose category filter intersects the subtree under
rootUuid.

return this.prisma.\$queryRawUnsafe(\`

SELECT tg.id as \"ruleId\", f.category_uuid as \"targetUuid\"

FROM tax_group_rules tg

JOIN tax_group_filters f ON f.rule_id = tg.id

WHERE f.category_uuid IN (SELECT uuid FROM categories WHERE path @\>
(SELECT path FROM categories WHERE uuid = \$1))

\`, rootUuid) as any

}

async buildRetargetPlan({

subtreeRootUuid,

newTargetUuid

}: { subtreeRootUuid: string, newTargetUuid: string }):
Promise\<RetargetPlan\[\]\> {

// Map each affected rule\'s target to the confirmed new parent (or
ancestor)

const affected = await this.affectedBySubtree(subtreeRootUuid)

if (!affected.length) return \[\]

const \[oldTargets, newTarget\] = await Promise.all(\[

this.prisma.category.findMany({ where: { uuid: { in: affected.map(a =\>
a.targetUuid) } }, select: { uuid: true, path: true } }),

this.prisma.category.findUnique({ where: { uuid: newTargetUuid },
select: { uuid: true, path: true } })

\])

const oldByUuid = Object.fromEntries(oldTargets.map(o =\> \[o.uuid,
o\]))

return affected.map(a =\> ({

ruleId: a.ruleId,

oldTarget: { uuid: a.targetUuid, path: oldByUuid\[a.targetUuid\]?.path
\|\| \[\] },

newTarget: { uuid: newTarget.uuid, path: newTarget.path }

}))

}

async applyRetarget(plan: RetargetPlan\[\]) {

if (!plan.length) return { updated: 0 }

// Bulk swap the category_uuid in rule filters

const ids = plan.map(p =\> p.ruleId)

const newUuid = plan\[0\].newTarget.uuid

await this.prisma.tax_group_filters.updateMany({

where: { ruleId: { in: ids } },

data: { categoryUuid: newUuid }

})

return { updated: ids.length }

}

}

const result = await this.svc.applyRetarget(plan)

return { plan, result }

}

}

## **C) "No Broken Blocks" Guard in the Assembler (NEW)**

**Explanation\**
If, after retargeting, a block would be empty (e.g., no items yet under
the new parent), switch to a **fallback** (e.g., roll up to parent or
"All" for that entity), so pages never render blank sections. This runs
server-side and doesn't alter your existing rule engine.

**Folder path\**
/apps/api/src/category-pages/assembler.guard.ts (new)

// /apps/api/src/category-pages/assembler.guard.ts

export type Block = { key: string; entity:
\'projects\'\|\'services\'\|\'teams\'\|\'talent\'\|\'guides\'\|\'courses\';
items: any\[\]; type: \'grid\'\|\'carousel\' }

export function ensureNoBrokenBlocks(blocks: Block\[\], fallback: (b:
Block) =\> Promise\<Block\>): Promise\<Block\[\]\> {

return Promise.all(blocks.map(async (b) =\> {

if (Array.isArray(b.items) && b.items.length \> 0) return b

// Try one fallback step (e.g., roll up one level or switch to "All"
taxonomy)

const fb = await fallback(b)

return (fb && fb.items && fb.items.length \> 0) ? fb : { \...b, items:
\[\] } // never crash

}))

}

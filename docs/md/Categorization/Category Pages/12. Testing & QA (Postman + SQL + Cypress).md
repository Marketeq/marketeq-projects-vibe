# **Testing & QA (Postman + SQL + Cypress)**

This doc covers the **Category Pages (L1/L2/L3)**. It tells you exactly
**what to test** and **where the tests live**.

## **Scope**

Verify the Category Pages (L1/L2/L3) per designs:

- **Separate pages** for **Show‑All / Projects / Services / Teams /
  Talent** (not tabs).

- **Suggestion modules:** Related Categories (semantic), Trending,
  Popular, Related Skills.

- **Published‑only** visibility on public pages.

- **Rule evaluator** correctness in blocks.

- **Pagination with cursors** (no duplicates).

- **SEO:** defaults + overrides (title/description, JSON‑LD breadcrumbs,
  canonical, OG/Twitter).

- **Selective revalidation** after relevant data changes.

- **Compatibility smoke checks** that the page payload reflects changes
  after webhook/cron --- **without calling other services directly**.

> **Important scope limit**: Tests must hit **only**
> /api/category-pages/:slug\[?kind=...\] and the **Next.js UI** that
> consumes it. **Remove** any Postman/Cypress calls straight to other
> upstream services.

## **Environments & Test Data**

- **Env vars:** STRAPI_URL, NEST_URL, FRONTEND_URL, DATABASE_URL.

- **Seed:** At least one full path **L1 → L2 → L3** with content for
  each entity (**Projects/Services/Teams/Talent**).

- **Publishing state:** Ensure suggested categories used in tests are
  **published**.

**1. Suggested categories are published**

****\-- Given a captured JSON payload per page under test

WITH payload AS (

SELECT resp #\> \'{suggestions}\' AS suggestions

FROM last_api_responses \-- capture of /api/category-pages

), ids AS (

SELECT (elem-\>\>\'uuid\')::uuid AS uuid

FROM payload, jsonb_array_elements(suggestions-\>\'relatedCategories\')
elem

UNION

SELECT (elem-\>\>\'uuid\')::uuid FROM payload,
jsonb_array_elements(suggestions-\>\'trendingCategories\') elem

UNION

SELECT (elem-\>\>\'uuid\')::uuid FROM payload,
jsonb_array_elements(suggestions-\>\'popularCategories\') elem

)

SELECT COUNT(\*) FILTER (WHERE c.published_in_marketplace = true) AS
published_count,

COUNT(\*) AS total

FROM categories c

JOIN ids ON ids.uuid = c.uuid;

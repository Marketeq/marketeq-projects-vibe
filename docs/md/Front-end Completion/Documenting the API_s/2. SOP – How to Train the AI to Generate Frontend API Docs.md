# **SOP -- How to Train the AI to Generate Frontend API Docs**

**Audience:** Docs Owner / Tech Lead / Backend-Frontend Bridge\
**Goal:** Generate clean, Stripe-style frontend API documentation
automatically by feeding the AI three inputs:\
1️⃣ The backend-exported openapi.json\
2️⃣ A short "UI Map" describing which endpoints belong to which
page/epic\
3️⃣ The **Frontend-Only API Template (Stripe-style example)**

## **1️⃣ Purpose & Overview**

**Explanation\**
This SOP explains how to "train" the AI so it knows *exactly* how to
format the frontend API documentation.\
\
You'll attach the three files above, paste a ready-made prompt, and
receive a fully formatted doc that frontend developers can use
immediately---clear, grouped by epic/page, with real example requests
and copy-paste snippets.

**Why it matters\**
This step saves hours of manual documentation work and ensures every
page follows the same, Stripe-like standard.

## **2️⃣ Prerequisites**

**Files you need before you start:**

  **File**                   **Source**                                                **Example Path**
  -------------------------- --------------------------------------------------------- ----------------------------------------------
  openapi.json               Generated by backend engineer (see Export & Enrich SOP)   apps/\<microservice\>/openapi/openapi.json
  UI_Map.txt                 Created by frontend lead or doc owner                     /docs/api/ui-maps/\<epic-or-page\>.txt
  frontend-api-template.md   Our Stripe-style template example (Document 2)            /docs/api/templates/frontend-api-template.md

**Explanation\**
The openapi.json provides raw endpoint data, the UI_Map.txt tells the AI
which endpoints appear on which screens, and the template shows it what
the finished doc should look like.

## **3️⃣ Prepare the UI Map**

**Explanation\**
The UI Map guides the AI on where each endpoint appears in the product.
It can be written in plain text---no strict format needed.

**Example (Checkout Epic)**

****Epic: Checkout

\- Checkout page (step 1): listPaymentMethods, getCart

\- Checkout page (step 2): createPaymentIntent, confirmPayment

\- Order confirmation page: getOrderById

**Hint\**
Keep it short (3--6 bullets). Just enough for the AI to understand
grouping.

## **4️⃣ Open the AI environment**

**Explanation\**
You can use ChatGPT (GPT-5), Claude, or your internal documentation
assistant---any model that accepts file uploads and large text prompts.

**Steps**

1.  Open a **new chat/session** to keep context clean.

2.  Click **Attach File / Upload** and attach:

    - openapi.json

    - UI_Map.txt

    - frontend-api-template.md

3.  Wait for confirmation that all files were uploaded successfully.

**Hint\**
Always upload all three together---this ensures the AI learns both
structure (template) and content (OpenAPI + UI Map).

## **5️⃣ Paste the training prompt**

**Explanation\**
This prompt teaches the AI exactly what to do with the three
attachments. Copy it verbatim into the message box after uploading the
files.

**Prompt**

****You are generating frontend-only API documentation for developers.

Inputs:

1\. openapi.json → full endpoint list for one microservice or feature

2\. UI_Map.txt → which endpoints belong to which page or epic

3\. frontend-api-template.md → example format to follow exactly

Output requirements:

\- Follow the exact structure, formatting, and tone of the attached
template example. Don\'t change anything in the format.

\- Group endpoints by epic/page as shown in the UI Map.

\- For each endpoint, include:

• Clear title and 1-sentence purpose

• \"Used in UI\" line from the UI Map

• Method and full URL (use \${NEXT_PUBLIC_API_BASE_URL}/v1 prefix)

• Auth type (Bearer/Public)

• Path & query params with examples

• Request body JSON (if applicable)

• Response JSON with realistic example data

• Common errors (HTTP code + short message)

• One copy-paste JS fetch() example

\- Use plain, friendly language (like Stripe docs).

\- Exclude backend-only or admin endpoints.

\- Output as clean Markdown, ready to drop into
/docs/api/frontend/\<epic-name\>.md.

**Hint\**
Keep the prompt short and structured---don't add commentary or extra
instructions.

## **6️⃣ Review the AI output**

**Explanation\**
Once the AI generates the document, review for three key things:

  -----------------------------------------------------------------------
  **Check**            **What to Look For**
  -------------------- --------------------------------------------------
  ✅ **Clarity**       Each endpoint has 1--2 sentence explanations, no
                       jargon.

  ✅ **Grouping**      Endpoints correctly grouped under the page/epic.

  ✅**Completeness**   All major frontend-used endpoints appear;
                       backend-only are omitted.
  -----------------------------------------------------------------------

**Hint\**
If grouping or tone feels off, paste this follow-up:

> "Regenerate the output following the uploaded frontend-api-template.md
> more strictly."

## **7️⃣ Save and commit the generated doc**

**Explanation\**
Once approved, save the output and add it to your Google Drive Folder.

**File path convention**

****/docs/api/frontend/\<epic-or-page-name\>.md

**Commit message example**

****docs(api): add Checkout frontend API documentation (AI-generated)

**Hint\**
Use one file per epic/page for clarity---e.g., checkout.md,
talent-profile.md, billing.md.

## **8️⃣ Common Troubleshooting**

  **Problem**                        **Likely Cause**                            **Fix**
  ---------------------------------- ------------------------------------------- ------------------------------------------------------
  AI merges backend-only endpoints   openapi.json not enriched or missing tags   Re-export OpenAPI with \@ApiTags or update filtering
  Missing endpoint groupings         UI Map incomplete                           Add bullet points for missing screens and rerun
  Output too verbose/technical       Template not attached                       Always attach the frontend-api-template.md

## **9️⃣ Expected Final Result**

**Explanation\**
After following this SOP, you'll have a new Markdown doc that looks and
reads like a Stripe API reference, but scoped to one frontend epic or
page.

**Example Output File (simplified)**

****\# Frontend API -- Checkout Flow

\## 1) Cart & Pricing

\### GET /v1/checkout/cart

Purpose: Fetch the current cart ...

\...

**Hint\**
Once generated, this doc becomes the source of truth for frontend devs
wiring endpoints into that epic.

## **10️⃣ Maintenance & Regeneration**

**Explanation\**
Whenever a backend engineer updates endpoints or adds new ones:

1.  Re-export openapi.json.

2.  Update UI_Map.txt (if pages changed).

3.  Re-run this SOP to refresh the AI-generated doc.

**Hint\**
Version the docs with the sprint or release tag (e.g., checkout-v2.md)
if major endpoint changes occur.

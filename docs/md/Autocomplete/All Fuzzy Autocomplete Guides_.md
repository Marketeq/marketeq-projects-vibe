**All Fuzzy Autocomplete Guides:**

**1. Fuzzy Autocomplete Developer QA Checklist**

Detailed test plan for validating the entire autocomplete system after
updates or new entries.

**2. Fuzzy Autocomplete Validation Utilities Guide**

Covers required libraries (spellcheck, profanity filter, category
matcher), with setup and examples.

**3. Fuzzy Autocomplete Auto-Approval Script Guide**

Step-by-step for building a Node.js-based script that automatically
approves valid suggestions.

**4. Fuzzy Autocomplete Logging & Metrics Guide**

Explains how to generate JSON logs from every validation run, what
metrics to track, and how to surface red flags.

**5. Fuzzy Autocomplete JSON Structure Reference**

Full schema for each entry type (skill, cert, job title), with field
explanations and sample data.

**6. Fuzzy Autocomplete GitHub Action Setup Guide**

CI/CD automation guide that runs validation on PR using GitHub Actions.

**7. Fuzzy Autocomplete Moderation Workflow**

Guide for human reviewers handling rejected or edge-case suggestions
(now edge-only).

**8. Fuzzy Autocomplete Edge Case Handling Guide**

Rules and patterns for judging borderline suggestions, slang terms, or
emerging roles.

**9. Fuzzy Autocomplete Onboarding Guide**

Orientation doc for new devs, moderators, or QA team members working
with autocomplete.

**10. Fuzzy Autocomplete Docs Index**

Master index linking to all docs in the system, with file structure
overview and Slack channels.

**11. Fuzzy Autocomplete Dev README**

Project README for the codebase with scripts, commands, and file
structure --- dev-facing.

**12. Fuzzy Autocomplete PR Template**

Pull request template for submitting new entries, validation fixes, or
logging updates.

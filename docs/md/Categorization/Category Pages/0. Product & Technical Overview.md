# **Product & Technical Overview**

- **Bi-directional authority**: Strapi is the primary **admin tool**,
  but not a single source of truth. Strapi → NestJS via webhooks; NestJS
  → Strapi via API. We rely on **UUID parity** and **timestamps** for
  conflict resolution ("last-writer-wins" with guardrails on
  slugs/UUIDs).

- **L2 are subcategories of L1; L3 are subcategories of L2**
  (facet/filter logic, breadcrumbs, URL resolution, and evaluators will
  respect this strict parentage).

- **Show-All + per-entity tabs at every level** (L1/L2/L3): first page
  is a combined grid; subsequent tabs are Projects-only, Services-only,
  Teams-only, Talent-only (design-driven).

- **Tax Groups ("rules") power dynamic sections** (Top-Rated Projects,
  Trending Teams, Newest Services, etc.). v1 includes **hybrid mode**:

  - Strapi switch enables **Manual Overrides**; manual items pin to the
    front, AI/rule-based content fills remaining slots.

  - When off, blocks are 100% rules/AI-driven.

- **Carousels vs Grids** are **design-driven**: if a section shows
  chevrons in the mock, we render a carousel; otherwise a grid.

- **Publishing thresholds** (admin policy): Categories must meet "full
  grid" before public publishing; **unpublished categories remain
  available to autocomplete and internal preview** (unchanged). For
  carousel blocks, **minimum content = 3× the visible grid size**, and
  **max scroll depth = 10** "pages" (no endless scroll).

- **Ranking signals** (blended): recency, rating, completion count,
  revenue, CTR, and admin boosts; **no sponsored yet** (future-ready).

- **SEO**: hybrid defaults + optional overrides; auto-generate from
  category path if blank (reuse existing logic).

- **Perf**: SSR + edge cache + incremental revalidation; webhook/cron
  invalidation; deterministic cache keys per (level, UUID path, TaxGroup
  hash).

- **Governance**: Only admins edit Tax Groups/Overrides in Strapi; we
  log changes alongside the reassignment plugin trail.
